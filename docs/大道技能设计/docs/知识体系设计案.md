# 知识体系设计案

## 1. 设计目标

知识体系不是另一套技能树，也不是单纯的经验条。它的目标是承接修仙游戏里的“理解、积累、领悟、传承”：

- **解释成长**：玩家不是凭空变强，而是对功法、术法、身体、神魂、世界规则理解更深。
- **支撑配置**：知识决定功法、技能、突破、事件和非战斗系统的准入门槛。
- **制造专精**：玩家不可能全部圆满，必须形成剑修、法修、体修、丹修、阵修等长期方向。
- **提供交叉**：高阶能力需要多类知识共同支撑，例如剑阵需要剑道和阵法，法宝需要炼器和神魂。
- **保留未来接口**：暂未实装的炼器、治理、御兽、天机等系统，可以先由知识作为配置入口。

一句话：

```text
知识 = 长期理解层
功法 = 知识增长和流派构筑层
技能 = 战斗或系统行为层
```

## 2. 核心原则

### 2.1 知识必须有用途

每个知识至少满足以下 2 条，才建议保留为独立节点：

- 解锁 2 个以上功法、技能、突破条件或事件选项。
- 加成 2 个以上功法、技能或系统效果。
- 属于某条流派的核心成长方向。
- 参与境界突破、道基、心魔、天劫等关键判定。
- 玩家能在 UI 上看懂它为什么重要。

如果只满足 0~1 条，应该合并、隐藏或暂缓投放。

### 2.2 知识不等于功能系统

有些知识暂时没有完整玩法系统，也可以存在，但不能强行展示给玩家刷。

例如：

- `world.trade` 在交易系统未完成前，可以作为事件选项门槛，暂不做独立成长目标。
- `artifact.smelting` 在炼器未完成前，可以只作为功法和未来配置前置。
- `heaven.omen` 在天机系统未完成前，可以只在特殊事件里出现。

未实装系统对应的知识应标记为“储备知识”或“不主动投放”，避免玩家看到一堆无用条目。

### 2.3 知识主要提供质变，不堆纯数值

知识等级 I~V 的意义：

| 等级 | 语义 | 设计用途 |
|---:|---|---|
| I | 入门 | 能学习低阶功法/技能，能理解基础事件 |
| II | 熟习 | 稳定使用，解锁常规构筑 |
| III | 小成 | 成为专业实用水平，出现第一次机制加成 |
| IV | 大成 | 作为跨境界、高阶技能和突破的常见门槛 |
| V | 圆满 | 解锁终局能力、特殊传承或高阶质变 |

默认每级可以有少量效率提升，但知识的主要价值应是：

- 解锁。
- 降低风险。
- 提高稳定性。
- 改变机制。
- 提供跨系统判断。

## 3. 知识分类

### 3.1 基础知识

基础知识是所有修士的底层能力，数量要少，使用面要广。

| 类型 | 示例 | 用途 |
|---|---|---|
| 呼吸感知 | 吐纳入门 | 修炼入门、功法学习、基础突破 |
| 经脉理解 | 经脉辨识 | 伤势、修炼稳定、突破风险 |
| 灵力控制 | 灵力操控 | 法术、御物、炼制、封技稳定 |
| 修真常识 | 修真博闻 | 鉴定、事件选项、低阶风险识别 |
| 道基理论 | 道基构筑 | 筑基、功法进阶、长期成长上限 |

基础知识建议在 UI 中常驻展示，因为它们能解释玩家整体修仙水平。

### 3.2 流派知识

流派知识服务战斗构筑和功法路线。

| 流派 | 知识方向 | 用途 |
|---|---|---|
| 剑修 | 剑气、兵器、剑心、剑阵 | 单体爆发、收割、破甲、阵地剑阵 |
| 法修 | 五行、雷火、遁法、领域 | 群攻、元素爆发、行动压制、领域 |
| 体修 | 骨血、淬体、恢复、法相 | 生存、反击、抗压、法相 |
| 符修 | 符纹、封印、咒法、契约 | 封技、标记、封疗、事件反制 |
| 神魂 | 神识、心防、分神、心魔 | 控制、感知、心劫、分身 |

流派知识应明显影响战斗技能和功法，否则玩家不会感受到“我在练这条路”。

### 3.3 生产知识

生产知识服务炼丹、炼器、阵法、符箓等系统。

如果对应系统未完成，生产知识可以先用于：

- 解锁相关功法。
- 解锁事件选项。
- 提供掉落识别。
- 作为未来技能和系统配置前置。

不要在系统未完成时大量投放生产知识经验，否则玩家会觉得练了没用。

### 3.4 世界知识

世界知识用于探索、地图、事件和资源选择。

| 知识 | 可用场景 |
|---|---|
| 草木 | 采集、炼丹材料识别、毒草风险 |
| 妖兽 | 敌人弱点、御兽事件、材料掉落 |
| 堪舆 | 地图节点、洞府、阵法地形 |
| 鉴定 | 物品信息、事件判断、交易防坑 |
| 贸易 | 市场、宗门经营、资源交换 |

世界知识的价值不一定体现在战斗，但必须体现在“做选择更聪明”。

### 3.5 境界知识

每个大境界建议保留 1~2 个总纲类知识：

| 境界 | 总纲作用 |
|---|---|
| 筑基 | 道基构筑、根基稳定 |
| 金丹 | 结丹总纲、丹火与核心控制 |
| 元婴 | 元婴理论、分神和重伤存续 |
| 化神 | 元神共鸣、领域雏形 |
| 渡劫 | 劫数推演、心劫和因果 |

境界知识是突破和高阶功法的硬门槛，不应该被普通战斗刷出来。

## 4. 知识状态

建议给每个知识增加设计状态，方便决定是否投放和展示：

| 状态 | 含义 | UI/投放策略 |
|---|---|---|
| `active` | 当前版本有明确用途 | 可展示、可成长、可作为奖励 |
| `support` | 作为前置或加成存在，但不是玩家主目标 | 可展示在相关详情中，不放到主成长目标 |
| `reserved` | 未来系统使用，目前未实装 | 不主动展示，不作为常规奖励 |
| `hidden` | 剧情、顿悟、特殊传承知识 | 触发后再展示 |

首版应该优先投放 `active`，谨慎投放 `support`，避免投放 `reserved`。

## 5. 知识成长来源

### 5.1 功法修炼

功法修炼是知识的主增长来源。

```text
修炼功法 -> 功法经验 -> 分配为功法包含知识经验
```

适合增长：

- 基础知识。
- 流派理论。
- 境界理论。
- 部分生产理论。

规则建议：

- 功法配置知识列表、成长权重、等级上限。
- 功法不能把知识提升到配置上限以外。
- 高阶功法可以继承低阶知识，但权重衰减。
- 实践型知识可以配置 `gainFromCultivation: false`。

### 5.2 战斗复盘

战斗复盘提供少量但高频的知识成长，用来制造“每场都变强”的感觉。

```text
战斗结束 -> 根据行为统计 -> 给相关知识少量经验
```

示例：

| 行为 | 增长知识 |
|---|---|
| 剑系伤害占比高 | 剑气、兵器 |
| 群攻命中多个敌人 | 五行、元素 |
| 护盾吸收大量伤害 | 经脉、炼体 |
| 封技成功 | 符箓、封印、灵力控制 |
| 多波战斗胜利 | 道基、战斗控制 |

限制：

- 低级敌人对高等级知识经验大幅衰减。
- 战斗复盘不能突破高阶知识关键门槛。
- 每场只奖励贡献最高的 1~3 类知识，避免乱涨。

### 5.3 实践行为

非战斗系统上线后，可以让操作行为提供知识经验。

| 行为 | 增长知识 |
|---|---|
| 炼丹成功/失败复盘 | 草木、药理、炉火 |
| 炼器锻造 | 材料、器纹、熔炼 |
| 布阵 | 阵纹、地脉、禁制 |
| 探索识别 | 鉴定、堪舆、妖兽 |
| 交易经营 | 贸易、势力、资源网络 |

实践行为适合增长“手艺类知识”，但不能完全替代功法和典籍。

### 5.4 典籍传承

典籍用于解锁方向和补课。

```text
获得典籍 -> 阅读/参悟 -> 获得知识经验、解锁上限或获得学习资格
```

典籍适合提供：

- 新知识 I 级入门。
- 低阶知识补课。
- 高阶知识上限解锁。
- 特殊分支知识。

典籍不建议直接把核心知识拉满，否则修炼和实践会失去意义。

### 5.5 顿悟事件

顿悟用于质变和惊喜，不作为稳定主来源。

示例：

- 残血反杀精英：心性、道心或体修知识获得大量经验。
- 第一次 1v5 多波无伤：身法、控制或战斗理解顿悟。
- 炼丹连续失败后成功：药理或炉火顿悟。
- 读残碑触发共鸣：天道、符文或古法知识解锁。

顿悟建议满足：

- 条件清晰但不强迫刷。
- 奖励稀有且有故事感。
- 多为突破瓶颈、解锁隐藏知识或提供大量经验。

## 6. 普通知识成长设计

普通知识包括基础理论、常识、境界理解、世界认知等。它们不应该要求玩家手动刷，而应自然增长。

推荐规则：

```text
功法修炼 = 主要来源
战斗/实践复盘 = 辅助来源
典籍/事件 = 补课和瓶颈突破
```

### 6.1 基础理论

基础理论通过通用功法和早期事件成长。

| 知识 | 成长来源 |
|---|---|
| 吐纳入门 | 基础吐纳诀、小周天、入门教学 |
| 经脉辨识 | 小周天、受伤恢复、医理事件 |
| 灵力操控 | 法术使用、符箓、御物、功法修炼 |
| 修真博闻 | 读书、探索、事件选择 |
| 道基构筑 | 筑基功法、突破准备、道基事件 |

### 6.2 境界理论

境界理论不能靠刷普通战斗成长，应主要来自：

- 对应境界功法。
- 突破准备。
- 师门讲法。
- 特殊典籍。
- 突破成功或失败后的复盘。

### 6.3 世界认知

世界认知通过探索和事件成长。

例如玩家在雾隐林反复采集、识别妖兽、处理异常地脉，就自然增长：

- 草木。
- 妖兽。
- 鉴定。
- 堪舆。

世界认知的奖励应体现在事件选择和信息透明度上，而不是直接加伤害。

## 7. 非战斗知识是否值得做

值得做，但要分阶段。

### 7.1 首版保留条件

首版非战斗知识只保留三类：

- 已经影响突破、功法或技能学习的。
- 即将接入炼丹、探索、地图事件的。
- 能作为奖励和事件判断的。

其他知识可以继续存在于配置中，但标记为 `reserved`，不主动投放，不出现在主 UI。

### 7.2 非战斗知识的意义

非战斗知识的意义不是让玩家多刷一条经验，而是：

- 让探索事件有不同解法。
- 让生产玩法有长期成长。
- 让掉落和地图选择更有判断。
- 让修仙世界不像只有打怪。
- 为后续宗门、炼器、天机、御兽等系统预留接口。

### 7.3 不值得做的情况

如果某个非战斗知识长期没有用途，也没有计划接入系统，就不值得展示。

处理方式：

- 合并到更大的知识。
- 作为隐藏前置，不显示给玩家。
- 标记为 `reserved`。
- 从首版奖励池移除。

## 8. 推荐首版知识池

首版建议只让玩家明显感知 25~40 个知识，不要一次展示全部 132 个。

### 8.1 必展示

| 类别 | 知识方向 |
|---|---|
| 基础 | 吐纳、经脉、灵力操控、修真博闻、道基 |
| 战斗 | 剑气、兵器、五行、雷火、身法、炼体、恢复、符箓封印 |
| 功法 | 周天、纯化、法力循环 |
| 突破 | 筑基总纲、结丹总纲 |

### 8.2 条件展示

| 类别 | 展示条件 |
|---|---|
| 炼丹 | 玩家获得炼丹功法、丹炉或相关任务 |
| 炼器 | 玩家获得炼器材料、法宝残片或炼器任务 |
| 阵法 | 玩家接触洞府、地图阵眼或阵法功法 |
| 符箓 | 玩家获得符纸、符册或封技技能 |
| 天道 | 玩家接触天劫、气运、因果事件 |

### 8.3 暂不展示

以下知识先作为未来接口：

- 宗门治理。
- 跨界贸易。
- 高阶洞天工程。
- 终局飞升航路。
- 复杂因果和命数系统。

## 9. UI 展示建议

知识 UI 不要一上来给玩家一整张巨大树。建议分三层：

### 9.1 当前相关

玩家当前装备功法、技能、突破目标关联的知识优先展示。

例如技能详情显示：

```text
破空剑气
受以下知识加成：
- 剑气 III：伤害 +12%
- 兵器 II：破甲 +4%
- 灵力操控 II：消耗 -3%
```

### 9.2 流派页

按流派展示关键知识：

- 剑修路线。
- 法修路线。
- 体修路线。
- 符修路线。
- 丹修路线。

### 9.3 全知识树

完整知识树作为高级页面，给愿意规划的玩家使用。默认不把 132 个节点全部压到新手脸上。

## 10. 配置字段建议

在现有 `dao_tree.yaml` 基础上，建议后续给知识补充几个设计字段：

```yaml
  - id: "foundation.control"
    name: "灵力操控"
    domain: "foundation"
    realm: "qi"
    rank: 2
    maxLevel: 5
    designState: "active"
    visibility: "always"
    growthSources:
      - "self_study"
      - "cultivation_method"
      - "combat_review"
      - "book"
      - "event"
    usageDomains:
      - "cultivation_method"
      - "combat_ability"
      - "breakthrough"
    rewardWeight: 1.0
    independentStudy:
      enabled: true
      maxLevel: 3
      efficiency: 1.0
```

字段说明：

| 字段 | 用途 |
|---|---|
| `designState` | `active`、`support`、`reserved`、`hidden` |
| `visibility` | `always`、`related`、`discovered`、`hidden` |
| `growthSources` | 允许哪些来源增长 |
| `usageDomains` | 已有字段，继续用于说明用途 |
| `rewardWeight` | 在复盘和事件奖励中的权重 |
| `independentStudy` | 自主学习开关、上限和效率修正 |

## 11. 数值建议

### 11.1 EVE 式训练速度与难度

自主学习采用类 EVE 的训练点模型：

```text
每日训练点 = 主属性 + 副属性 / 2
升到目标等级所需训练点 = basePoints × rank × levelMultiplier[targetLevel]
学习天数 = 剩余训练点 ÷ 每日训练点
```

当前配置对应：

- `basePoints = 250`
- `levelMultipliers = [1, 2, 4, 8, 16]`
- `rank` 越高，知识越难学。
- 大道配置里的 `primary` / `secondary` 决定使用哪些基础属性。

示例：

```text
修真通识：primary = 悟性，secondary = 心性
若悟性 12，心性 10，则每日训练点 = 12 + 10 / 2 = 17
rank 1 知识升 I 需要 250 点，约 15 日
rank 3 知识升 I 需要 750 点，约 45 日
```

基础属性只影响学习速度，不直接给知识等级。难度由 `rank` 和等级倍率控制。

自主学习定位为补课和冲门槛，默认最多学到 III。高阶天道类知识默认最多 I，境界总纲/道果类默认最多 II；想继续到 IV/V，需要功法、传承、顿悟或系统实践。

### 11.2 来源比例

知识经验来源比例：

| 来源 | 占比 | 说明 |
|---|---:|---|
| 功法修炼 | 60%~70% | 稳定主增长 |
| 战斗/实践复盘 | 15%~25% | 高频反馈 |
| 典籍传承 | 5%~15% | 解锁和补课 |
| 顿悟事件 | 0%~10% | 稀有质变 |

经验限制：

- 低阶行为不能无限刷高阶知识。
- `reserved` 知识不进入常规奖励池。
- `support` 知识只在相关行为中少量增长。
- 境界总纲类知识主要靠功法、典籍、突破事件成长。
- 每场战斗最多奖励 1~3 个知识，避免奖励信息噪音。

## 12. 首版落地步骤

1. 给全部知识标记 `designState`。
2. 筛出首版 `active` 知识，数量控制在 25~40 个。
3. 每个 `active` 知识至少绑定 2 个用途。
4. 给功法配置知识成长权重和上限。
5. 给战斗结算配置 5~8 类复盘奖励。
6. 给典籍和事件配置补课奖励。
7. UI 只展示当前相关知识和已发现知识。

## 13. 验收标准

知识体系首版完成后，应满足：

- 玩家知道当前修炼会提升哪些知识。
- 玩家知道某个技能或功法为什么需要某些知识。
- 普通知识能自然增长，而不是要求玩家手动刷。
- 非战斗知识不会在没有用途时打扰玩家。
- 至少 3 个战斗流派能通过知识产生明显差异。
- 至少 1 个非战斗方向能通过知识影响事件或奖励。
- 回看旧内容时，玩家能感到“我现在懂得更多，所以选择更多、风险更低”。

最终目标不是让玩家把 132 个知识都练满，而是让每个被看见的知识都有理由存在。
