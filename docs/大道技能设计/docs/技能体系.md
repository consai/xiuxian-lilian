# 技能体系

## 1. 核心关系

- **知识**：角色永久掌握的理论与能力，决定技能能否学习，并加成技能效果。
- **功法**：增长部分知识并提供功法属性。
- **技能**：战斗行为或通用被动，不负责直接代表全部知识。

同一知识可以加成多个技能，同一技能也可以混合多项知识。

## 2. 技能类型

仅保留四种技能：

| 类型 | 生效方式 |
|---|---|
| `combat_active` | 装入战斗技能栏后主动施放 |
| `combat_passive` | 装入战斗被动栏后生效 |
| `combat_upkeep` | 主动开启并持续支付资源后生效 |
| `general_passive` | 学会后永久生效，由效果类型分发至生产、探索、治理等系统 |

突破、炼器行为、炼丹配方、阵法布置、采集和治理指令不属于技能；它们由各自系统处理，
通用被动只为这些系统提供属性加成。

## 3. 技能学习

每个技能自身配置：

- 最低境界 `learningRequirements.realm`
- 可选知识等级门槛 `learningRequirements.knowledge`

普通技能不配置知识学习门槛，达到境界并获得技能来源后即可学习。只有稀有以上的特殊技能、终局技能或明确需要传承理解的技能，才允许配置知识门槛。

知识学习门槛护栏：

- `common` / `uncommon` 技能：`learningRequirements.knowledge` 必须为空。
- `rare` 及以上技能：最多 2 项知识门槛。
- 门槛等级最高为 II 级，不用 III–V 级卡学习。
- 更深的理解放在 `knowledgeScaling` 中影响效果值成长，不作为学习硬门槛。

学习流程：

```text
获得技能学习机会 -> 检查技能自身门槛 -> 永久解锁技能
```

本次设计不定义技能来源、掉落与交易系统。

## 4. 知识加成

每个技能配置若干 `knowledgeScaling.knowledge`：

- `requiredLevel`：学习技能所需的最低知识等级。
- `scalingWeight`：该知识对技能效果的相对权重。

```text
单项掌握率 = clamp((知识有效等级 - requiredLevel) / (5 - requiredLevel), 0, 1)
综合掌握率 = Σ(单项掌握率 × normalized(scalingWeight))
最终效果 = 技能基础效果 + knowledgeGrowth × 综合掌握率
```

刚满足学习门槛时不获得知识成长值。相关知识全部达到 V 级时，技能获得完整
`knowledgeGrowth`；固定值与百分比效果分别受自身上下限约束。

## 5. 战斗技能

战斗主动配置目标、显式攻击倍率 `powerScale`、施法时间、冷却与单次消耗。首版运行时暂不结算
施法时间，但保留字段供后续接入。

战斗被动必须装备后才生效，不具有施法时间与资源消耗。

战斗持续消耗技能开启后提供领域、法相或特殊状态，并持续支付 `upkeepCostsPerSecond`；
无法支付时自动关闭。

## 6. 通用被动

通用被动学会后永久生效。同一 `stackGroup` 的通用被动效果只取最高值。运行时根据 `effectId`
将效果分发到对应系统，例如：

- `craft_quality` -> 炼器系统
- `pill_quality` -> 炼丹系统
- `formation_power` -> 阵法系统
- `appraisal_accuracy` -> 鉴定系统
- `settlement_output` -> 宗门治理系统

通用被动不进入战斗技能栏，也不配置 `combat` 字段。

## 7. 统一效果目录

技能和功法共同引用 `config/effect_catalog.json`。所有效果必须登记稳定 ID、目标类型、数值类型和
默认叠加策略。

正面效果使用 `scalingMode: positive`；负面与削弱效果使用 `scalingMode: magnitude`，知识提升时
增强其绝对幅度，并通过 `clampMin`、`clampMax` 限制最终范围。
