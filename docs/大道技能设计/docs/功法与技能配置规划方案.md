# 功法与技能配置规划方案

## 1. 总目标

功法和技能要共同服务三个体验：

1. **每次修炼都变强**：功法提供长期属性、知识经验和流派方向。
2. **每场战斗都能验证变强**：技能提供可见的伤害、控制、续航和清场表现。
3. **每个流派都有打法差异**：同样是 1v5 多波战斗，剑修、法修、体修、符修、阵修的解法不同。

配置关系固定为：

```text
知识 -> 解锁/强化功法和技能
功法 -> 增长知识 + 提供常驻构筑属性
技能 -> 战斗中执行具体行为
效果目录 -> 统一描述属性、伤害、控制、资源、生产和探索加成
```

不要让功法和技能互相抢职责：

- 功法决定“我是什么流派，我长期强在哪”。
- 技能决定“我这一场战斗怎么出手”。
- 知识决定“我对这个体系理解有多深，能不能产生质变”。

## 2. 玩家配置结构

### 2.1 功法槽位

建议保留现有 4 个功法槽位：

| 槽位 | 数量 | 作用 |
|---|---:|---|
| 主功法 | 1 | 决定核心流派、修炼速度、主资源恢复和主要战斗加成 |
| 辅助功法 | 2 | 补足生存、资源、控制、生产或探索能力 |
| 身法 | 1 | 提供闪避、先手、速度、脱战和特殊移动效果 |

主功法必须影响战斗手感。否则玩家修炼半天，只在面板上看到数字，会很弱。

主功法至少配置 3 类效果：

- 核心攻击或防御：例如剑伤、法伤、气血、减伤。
- 资源循环：例如法力恢复、体力恢复、冷却效率。
- 流派特性：例如击杀回蓝、群攻加成、低血护盾、封技强化。

辅助功法用于横向构筑，不应强过主功法：

- 补续航：青木、归息、丹医。
- 补防御：金刚、护体、心防。
- 补控制：符箓、神魂、阵法。
- 补非战斗收益：炼丹、炼器、探索、经营。

身法不只是移速。它应提供战斗节奏价值：

- 闪避窗口。
- 开局先手。
- 被围攻时减伤。
- 绕开集火或制造后排暴露窗口。
- 低血脱困。

### 2.2 技能槽位

建议首版使用：

| 槽位 | 数量 | 作用 |
|---|---:|---|
| 主动技能 | 5 | 自动战斗的核心按钮 |
| 战斗被动 | 2 | 改变流派规则 |
| 持续技能 | 1~2 | 护体、剑阵、领域、法相等开关型能力 |
| 法宝 | 2 | 大冷却、高辨识度的战斗工具 |
| 道具 | 2 | 丹药、符箓、应急资源 |

5 个主动技能推荐形成固定功能配比：

| 槽位定位 | 目的 | 示例 |
|---|---|---|
| 填充输出 | 冷却短、保证出手频率 | 御气弹、普通剑气 |
| 爆发/收割 | 点杀低血或精英 | 破空剑气、金丹真火 |
| 群攻 | 处理 3~5 个敌人 | 火鸦术、五行轮爆 |
| 控制/封技 | 反制治疗者、召唤者、控制者 | 缚灵符、雷音震魄 |
| 生存/回复 | 多波续航 | 凝气护身、玄元归息 |

玩家可以换流派，但这个功能配比要作为推荐配置存在。

## 3. 功法配置方案

### 3.1 功法字段规划

每个功法层建议保持以下字段：

```yaml
- id: "method.taixu_sword.1"
  familyId: "method.taixu_sword"
  tier: 1
  realm: "qi"
  role: "剑修入门，单体爆发与击杀回转"
  rarity: "epic"
  predecessorId: ""
  nextMethodId: "method.taixu_sword.2"
  knowledgeXpRatio: 0.65
  practice:
    baseEfficiency: 1.0
    dayCost: 1
  knowledge:
    - skillId: "sword.qi"
      capLevel: 2
      growthWeight: 3
      masteryWeight: 3
      gainFromCultivation: true
    - skillId: "foundation.control"
      capLevel: 2
      growthWeight: 1
      masteryWeight: 1
      gainFromCultivation: true
  effects:
    - effectId: "damage_sword"
      base: 0.08
      masteryGrowth: 0.06
      operation: "add_percent"
      stackGroup: "method_sword_damage"
      stackPolicy: "highest"
      activation: "active_method"
      cap: 0.6
  learningRequirements:
    realm: "qi"
    knowledge: []
```

普通功法和传承首层不配置知识学习门槛。特殊功法层才允许配置 1~2 项知识门槛，且最高只到 II 级。

### 3.2 功法层级规划

每个完整传承建议 9 层，对应大境界：

| 层 | 境界 | 配置重点 |
|---:|---|---|
| 1 | 练气 | 建立流派基础，给一个明显战斗特性 |
| 2 | 筑基 | 补足资源循环或生存短板 |
| 3 | 金丹 | 出现第一个强质变，例如阵、真火、剑心 |
| 4 | 元婴 | 引入分身、法相、神魂或领域雏形 |
| 5 | 化神 | 强化战场规则，例如大范围压制 |
| 6 | 炼虚 | 引入空间、虚实、洞天类效果 |
| 7 | 合体 | 流派完整体，技能联动明显增强 |
| 8 | 大乘 | 终局构筑，强但资源压力大 |
| 9 | 渡劫 | 飞升前专精，反制天劫和首领机制 |

每层功法不要只加数值。推荐每 2 层给一次机制变化：

- 练气：获得流派标识。
- 筑基：资源循环成型。
- 金丹：战斗内第一次质变。
- 元婴：获得第二行动源，分身/法相/领域。
- 化神以后：改变敌人规则或波次规则。

### 3.3 功法流派模板

| 流派 | 主功法定位 | 核心效果 | 短板 | 推荐搭配 |
|---|---|---|---|---|
| 剑修 | 单体爆发、击杀回转 | 剑伤、破甲、击杀减 CD | 被 5 人围攻时压力大 | 群攻法术、护体功法 |
| 法修 | 群攻、元素爆发 | 法伤、群体伤害、法力效率 | 低蓝时疲软 | 回蓝功法、控制技能 |
| 体修 | 生存、反击、稳定推进 | 气血、减伤、韧性、反伤 | 清怪较慢 | 群攻技能、法宝 |
| 符修 | 封技、封疗、标记引爆 | 减速、易伤、封疗、延后特殊行动 | 纯伤害低 | 爆发技能、剑修辅助 |
| 阵修 | 多波长线、区域压制 | 周期伤害、敌方减速、波次增益 | 启动慢 | 防御技能、回蓝 |
| 丹修 | 续航、恢复、状态净化 | 回复、护盾、负面清除 | 爆发不足 | 剑修/法修技能 |
| 神魂 | 控制、心防、精神伤害 | 恐惧、混乱、行动干扰 | 对无心智敌人弱 | 物理/法术补伤 |

## 4. 技能配置方案

### 4.1 技能字段规划

每个技能建议保持已有结构，并补足“自动战斗可理解”的标签和触发建议：

```yaml
- id: "ability.combat.five_phase_burst"
  name: "五行轮爆"
  type: "combat_active"
  realm: "foundation"
  rarity: "rare"
  description: "引动五行灵力，对当前场上敌人造成群体伤害。"
  tags:
    - "spell"
    - "area"
    - "wave_clear"
  knowledgeScaling:
    knowledge:
      - skillId: "spell.five_phase"
        requiredLevel: 2
        scalingWeight: 3
      - skillId: "spell.elements"
        requiredLevel: 1
        scalingWeight: 1
  combat:
    target: "enemies_all"
    castTime: 1.0
    cooldown: 10
    costs:
      - resource: "mana"
        value: 24
    upkeepCostsPerSecond: []
    activation: "cast"
    powerScale: 0.65
  effects:
    - effectId: "damage_elemental"
      base: 46
      knowledgeGrowth: 24
      operation: "add_flat"
      target: "enemies_all"
      stackGroup: "damage_elemental"
      stackPolicy: "ability_instance"
      scalingMode: "positive"
  trigger:
    recommendedWhen:
      enemyCountGte: 3
      hasPriorityEnemy: false
      playerHpLte: null
  upgrade_options: []
  evolution_conditions: []
```

### 4.2 主动技能配置原则

主动技能必须写清楚 5 件事：

- **目标**：打谁。
- **消耗**：花什么资源。
- **冷却**：多久一次。
- **效果**：造成什么结果。
- **推荐触发**：自动战斗什么时候用。
- **学习门槛**：普通技能不配置知识门槛，特殊技能最多 2 项且最高 II 级。

推荐目标枚举：

| target | 用途 |
|---|---|
| `enemy` | 当前可选单体 |
| `enemy_lowest_hp` | 当前可选目标中的低血收割 |
| `enemy_front` | 打前排、破防 |
| `enemy_priority` | 当前可选目标中的高威胁敌人 |
| `enemies_all` | 全体群攻 |
| `self` | 防御、回复、增益 |
| `area` | 阵法、领域 |

后排默认受前排保护，不能被主动技能直接指定。想处理后排，需要先满足至少一种条件：

- 前排敌人全部死亡或被击退，后排自动进入可选目标集合。
- 后排单位被标记为高威胁时，仍需要先暴露或通过群攻、穿透、阵法、领域间接处理。
- 后排单位被标记为 `exposed`，短时间内可被单体技能选中。
- 群攻、穿透、溅射、阵法、领域等效果可以间接影响后排，但单体爆发不能无条件越过前排。

`enemy` 和 `enemy_lowest_hp` 只从当前可选目标中选择，不会默认越过前排锁定后排。

### 4.3 被动技能配置原则

战斗被动不要配置成普通属性 +5%。它应该改变战斗规则：

| 被动方向 | 例子 |
|---|---|
| 击杀奖励 | 击杀后回蓝、缩短冷却、获得护盾 |
| 低血反转 | 血量低于 35% 自动减伤或反击 |
| 多敌增益 | 敌人数越多，群攻或防御越强 |
| 标记联动 | 首次命中施加印记，后续技能引爆 |
| 波次收益 | 每进入新波，获得短暂护盾、回蓝或攻击提升 |

### 4.4 持续技能配置原则

持续技能适合承载“修仙大招形态”：

| 类型 | 例子 | 开启条件 |
|---|---|---|
| 护体 | 护体真气 | 玩家低血或敌人数多 |
| 阵法 | 三才剑阵 | 敌人数 >= 3 |
| 领域 | 小五行领域 | 有精英、召唤者、治疗者或控制者 |
| 法相 | 法相显化 | Boss 或最终波 |

持续技能必须有资源压力：

- 开启强，但不能一直无脑开。
- 资源不足时自动关闭。
- 自动战斗要有明确开关条件。

## 5. 成长与质变规划

### 5.1 技能等级质变

每个战斗技能建议 5 级，至少 2 个等级产生机制变化：

| 等级 | 成长类型 |
|---:|---|
| I | 解锁基础效果 |
| II | 数值提升，冷却或消耗小优化 |
| III | 第一次机制变化，例如溅射、破甲、护盾 |
| IV | 强化联动，例如击杀返还、封技回蓝 |
| V | 第二次机制变化，例如全体、连锁、领域化 |

示例：破空剑气

| 等级 | 效果 |
|---:|---|
| I | 对单体造成剑伤 |
| II | 伤害提升 |
| III | 对低血目标额外伤害 |
| IV | 击杀返还 40% 冷却 |
| V | 击杀后追加攻击另一个低血敌人 |

### 5.2 功法熟练度等级

功法熟练度为等级制，暂定每部功法 5 级。当前等级经验满后进入下一等级，最高 V 级。

功法效果类型固定，不随熟练度等级解锁或替换。熟练度等级只改变既有效果的数值，避免同一部功法在不同等级下出现两套效果逻辑。

| 等级 | 数值倍率 |
|---:|---:|
| I | `base + masteryGrowth * 0` |
| II | `base + masteryGrowth * 0.25` |
| III | `base + masteryGrowth * 0.5` |
| IV | `base + masteryGrowth * 0.75` |
| V | `base + masteryGrowth * 1` |

配置上建议拆成两个字段：

```yaml
mastery:
  maxLevel: 5
  level: 1
  currentXp: 0
  requiredXp: 100
```

配置层只写固定 `effects`，不要给熟练度等级单独配置新增效果。想要新机制，应放到下一层功法、技能、法宝或特殊传承中。

## 6. 境界投放规划

### 6.1 练气期

目标：让玩家学会基本配置。

推荐投放：

- 2 个填充输出：御气弹、普通剑气。
- 1 个小群攻：火鸦术。
- 1 个保命：凝气护身。
- 1 个身法：流风步。
- 2~3 本入门功法：基础吐纳诀、小周天行气法、御剑诀。

玩家体验：从“打一只怪都慢”到“能稳定过 2~3 敌小波”。

### 6.2 筑基期

目标：形成流派分化。

推荐投放：

- 剑修收割：破空剑气、剑心通明。
- 法修群攻：五行轮爆。
- 体修抗压：震血拳、护体真气。
- 符修反制：缚灵符。
- 主功法进入第一批传承层：太虚剑典、金刚不灭身、赤霄雷火典。

玩家体验：第一次明确感觉“我这套构筑有打法”。

### 6.3 金丹期

目标：多波战斗稳定成型。

推荐投放：

- 群体压制：雷音震魄。
- 波间续航：玄元归息。
- 持续阵法：三才剑阵。
- 高爆发：金丹真火。
- 每个主流派至少获得 1 个被动质变。

玩家体验：开始能处理 4~5 敌同场，能打精英波。

### 6.4 元婴以后

目标：从“技能强”升级到“规则强”。

推荐投放：

- 元婴出窍：制造暴露窗口和双线输出。
- 小五行领域：改变敌方行动速度和元素伤害。
- 法相显化：终局抗压形态。
- 万剑归宗：清场爆发。
- 因果锁命：首领专精。

玩家体验：高阶修士不是数字更大，而是能改变战场。

## 7. 自动战斗配置规划

自动战斗需要理解技能用途。建议每个技能配置 `tags` 和 `trigger.recommendedWhen`。

### 7.1 标签规范

| 标签 | 含义 |
|---|---|
| `filler` | 填充输出 |
| `burst` | 爆发 |
| `execute` | 收割 |
| `area` | 群攻 |
| `seal` | 封技/延后特殊行动 |
| `defense` | 防御 |
| `heal` | 回复 |
| `resource` | 回蓝/回体 |
| `upkeep` | 持续开关 |
| `wave_sustain` | 多波续航 |
| `boss` | 首领对策 |

### 7.2 三种预设

均衡：

```text
封技高威胁目标 -> 低血收割 -> 血低防御/回复 -> 敌多群攻 -> 爆发 -> 填充
```

进攻：

```text
低血收割 -> 敌多群攻 -> 爆发 -> 封技 -> 填充 -> 防御
```

保守：

```text
血低防御/回复 -> 封技 -> 资源恢复 -> 敌多群攻 -> 收割 -> 填充
```

## 8. 数值规划

### 8.1 技能数值

| 技能类型 | 冷却 | 单目标价值 | 备注 |
|---|---:|---:|---|
| 填充输出 | 2.5~4 秒 | 1.0 倍 | 低消耗 |
| 单体爆发 | 5~8 秒 | 1.6~2.2 倍 | 用于点杀 |
| 群体技能 | 8~14 秒 | 每目标 0.45~0.75 倍 | 敌越多越赚 |
| 封技技能 | 10~16 秒 | 0.4~0.8 倍 | 价值在控制 |
| 防御技能 | 8~18 秒 | 抵消 1.5~2.5 次普攻 | 应对集火 |
| 回复技能 | 12~24 秒 | 回复 15%~30% 关键资源 | 多波续航 |

### 8.2 功法数值

同境界主功法效果建议：

| 效果类型 | 常规范围 |
|---|---:|
| 单属性伤害加成 | 8%~18% |
| 生存加成 | 8%~20% |
| 资源恢复 | 5%~15% |
| 冷却效率 | 4%~12% |
| 特定场景加成 | 10%~25% |

功法反哺上限已有 `masteryBonusMax = 0.5`，可以继续保留。也就是知识全满时，本层功法效果最高约提升 50%。

## 9. 配置落地顺序

### 第一阶段：让战斗先有手感

1. 整理 20 个首版战斗技能。
2. 每个技能补齐 `tags`、`target`、`trigger`。
3. 保证 5 个主动槽能组成完整配置。
4. 做 3 套默认推荐：剑修、法修、体修。

优先技能：

- 御气弹
- 破空剑气
- 火鸦术
- 五行轮爆
- 缚灵符
- 凝气护身
- 护体真气
- 玄元归息
- 雷音震魄
- 三才剑阵

### 第二阶段：让功法决定流派

1. 每个主流派先做练气、筑基、金丹 3 层。
2. 每层至少 3 个效果。
3. 每个流派绑定 2~4 个核心知识。
4. 功法效果必须能强化对应技能。

优先功法：

- 基础吐纳诀：新手通用。
- 小周天行气法：资源循环。
- 太虚剑典：剑修。
- 赤霄雷火典：法修。
- 金刚不灭身：体修。
- 基础符册/清心定念法：控制与心防。

### 第三阶段：做成长质变

1. 技能等级 I~V 增加机制变化。
2. 功法熟练度改为 I~V 级，每级只提高固定效果的数值。
3. 战斗结算展示本场成长数据。
4. 旧区域保留低级敌人，让玩家回头能碾压。

### 第四阶段：扩展高阶规则

1. 增加领域、法相、剑阵、分身。
2. 增加精英和首领机制。

## 11. 验收标准

配置完成后，至少要能验证：

- 不同主功法会让同一套技能表现不同。
- 不同技能配置能明显改变 1v5 多波战斗结果。
- 玩家升技能后能看到机制变化，而不是只有伤害增加。
- 玩家修炼功法后，战斗用时、承伤、回蓝、清场速度至少有一项明显改善。
- 自动战斗三种预设使用同一套技能时，表现有差异。
- 回到旧区域时，玩家能明显碾压过去觉得危险的敌人。

一句话：功法负责“长期变强的方向”，技能负责“战斗里看见变强的瞬间”。两者都要配质变点，玩家才会觉得修炼真的有用。
