# 大道知识树设计规格

## 1. 核心定位

这不是传统“升级后点天赋”的树，而是一套 EVE 式长期知识训练系统：

- 每门知识独立训练 I–V 级，训练期间不要求玩家在线。
- 高级知识依赖多条跨领域前置，境界只是准入门槛，不会自动赠送能力。
- 玩家不可能在合理周期内学满全部知识，必须形成职业、战斗与生产专精。
- 战斗、生产、探索获得的是资源、见闻与突破条件；知识训练提供稳定、可预测的长期成长。

机器可读的完整节点和前置关系位于 `config/dao_tree.json`。该文件是本设计项目的最终交付物，
Godot 项目应直接读取此配置，不依赖本仓库中的校验脚本。

## 2. 九大境界

| 境界 | 系统含义 | 主要问题 |
|---|---|---|
| 练气 | 学会感知、储存、操纵灵气 | 生存、基础术法、选择方向 |
| 筑基 | 建立稳定且可扩展的修行架构 | 道基品质、专业入门 |
| 金丹 | 获得高密度生命与能量核心 | 法宝、丹火、规模化能力 |
| 元婴 | 拥有第二生命核心与远程投射能力 | 分神、跨域、重伤存续 |
| 化神 | 元神与天地法理共鸣 | 领域、法相、器灵 |
| 炼虚 | 处理虚实、空间和世界边界 | 虚空生存、空间能力 |
| 合体 | 统合肉身、元神、法相、洞天与法则 | 自成体系、势力治理 |
| 大乘 | 将毕生道路凝成完整道果 | 世界级影响、应劫准备 |
| 渡劫 | 面对天道对生命层次跃迁的总清算 | 劫雷、心劫、因果重构 |

境界突破要求：

1. 达到该境界的累计知识点门槛。
2. 将上一境界的“总纲”训练至 IV。
3. 完成至少一条专业大道的对应阶段知识 III。
4. 完成突破事件；失败造成损失，但不永久摧毁角色。

| 大道 | 定位 | 典型终局身份 |
|---|---|---|
| 修真通识 | 所有境界和系统的公共主干 | 飞升者 |
| 功法周天 | 法力容量、恢复、转化和功法兼容 | 法力海、功法宗师 |
| 炼体血脉 | 生存、恢复、近战与环境承载 | 不灭道身 |
| 神魂心境 | 感知、心防、分神、元神和心劫 | 元神圣者 |
| 术法五行 | 元素、遁术、领域和规则术式 | 神通道君 |
| 兵道剑道 | 武器、兵意、剑域和极致杀伐 | 剑道至尊 |
| 炼器机关 | 法宝、傀儡、飞舟和仙器 | 炼器大宗师 |
| 丹道医理 | 药理、医术、毒、生命与渡劫丹 | 丹圣 |
| 阵法禁制 | 洞府、防御、传送、空间与世界阵 | 阵道宗师 |
| 符箓咒法 | 储存术式、封印、契约和因果敕令 | 符诏天师 |

## 4. EVE 式训练规则

每门知识包含：

- `rank`：训练倍率，越高越慢。
- `maxLevel: 5`：固定五级。
- `prereqs`：前置知识及所需等级。
- `realm`：允许开始训练的最低境界。
- `domain`：所属大道，用于属性加成与导师加成。

JSON 顶层结构：

| 字段 | 内容 |
|---|---|
| `schemaVersion` | 配置结构版本，用于 Godot 导入兼容 |
| `metadata` | 配置名称、说明和知识总数 |
| `training` | 等级上限、基础训练点、倍率与公式说明 |
| `attributes` | 五项训练属性 ID 与显示名称 |
| `realms` | 境界顺序和累计知识点门槛 |
| `domains` | 大道定义、说明及主次训练属性 |
| `skills` | 全部知识节点、境界、倍率和前置 |

Godot 读取示例：

```gdscript
var text := FileAccess.get_file_as_string("res://config/dao_tree.json")
var dao_tree: Dictionary = JSON.parse_string(text)
```

单级训练点：

```text
训练点 = 250 × rank × [1, 2, 4, 8, 16][目标等级]
训练速度/分钟 = 主属性 + 次属性 ÷ 2
训练时间 = 训练点 ÷ 训练速度
```

等级语义统一：

| 等级 | 含义 |
|---|---|
| I | 能使用，失败率较高 |
| II | 能稳定使用 |
| III | 专业实用水平，可解锁多数后续 |
| IV | 精通水平，是跨境界与高阶知识的常见前置 |
| V | 理论圆满，通常解锁终局能力并提高效率上限 |

每级默认给予约 5% 效率提升，但知识的主要价值是解锁新能力，而非堆叠纯数值。

## 5. 属性与角色差异

五项训练属性只影响学习速度，不直接决定战斗数值：

- 悟性：理论、术法、制造、阵法。
- 神识：神魂、精密操控、符箓、阵法。
- 根骨：功法、炼体、武器承载。
- 心性：心防、长期修持、因果与渡劫。
- 福缘：探索、资源、丹道与天机。

允许有限次数的属性重置，避免新手永久误配。装备与丹药可短期改变训练速度，但不能绕过前置。

## 6. 专精与交叉前置

大道不是十二条互不相干的直线。关键高阶能力必须交叉：

- 剑阵需要剑道与阵法。
- 本命法宝需要炼器与神魂。
- 虚空飞舟需要炼器、器灵和空间阵理。
- 道丹需要丹道与法则种子。
- 掌中天地需要术法与洞天营造。
- 欺天渡劫阵需要星斗阵与天劫推演。
- 最终飞升同时要求道果、劫体与心劫超脱。

这确保生产玩家、探索玩家和战斗玩家互相依赖，也避免单一路线包办全部玩法。

## 7. 突破、死亡与失败

- 突破失败不删除已训练知识；损失突破材料、状态和部分未固化修为。
- 肉身死亡后，可依据元婴、元神、魂契、洞天等能力决定恢复方式。
- 心魔、丹毒、功法冲突、业力和法宝损毁均是可管理的长期风险。
- 渡劫不是单次数值检定，而是雷劫、心劫、因果劫和外敌干扰组成的终局事件。

## 8. 内容覆盖清单

完整树覆盖修士从练气至渡劫可能面对的主要问题：

- 修炼：纳气、周天、纯化、瓶颈、突破、功法冲突、仙灵转化。
- 生存：伤病、毒、心魔、夺舍、绝灵区、虚空、重伤与死亡恢复。
- 战斗：武器、术法、遁法、领域、法相、法则、劫数神通。
- 生产：丹药、法宝、傀儡、飞舟、符箓、阵法、洞天和仙器。
- 社会：交易、契约、宗门治理、资源网络和跨界文明。
- 探索：采集、鉴定、堪舆、天候、界域、虚空和飞升航路。
- 天道：气运、业力、命数、因果、法则、道果、天劫和飞升。

## 9. 平衡护栏

- 不允许用境界压制替代技能构筑；低境界专精者应能在专业领域胜过高境界外行。
- 高阶节点主要扩大选择和改变规则，避免单纯指数增伤。
- 所有强力能力至少拥有资源成本、准备时间、暴露风险、反制手段中的两项。
- 生产与后勤知识必须具有真实需求，不能由 NPC 无限供应替代。
- 终局技能的前置应跨越三条以上大道，防止孤立冲刺。
