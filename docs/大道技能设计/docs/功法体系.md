# 功法体系设计规格

## 1. 功法与知识的关系

知识是角色永久掌握的能力，功法是增长和运用知识的方法。

- 修炼功法会同时增长功法熟练度与其包含的知识。
- 每项知识具有 `capLevel`，功法不能把该知识提升到此等级以上。
- 功法可以只教授一项知识，也可以按权重混合多项知识。
- 玩家已有的知识掌握度会反向提高功法效果。
- 普通功法学习不检查知识门槛；特殊功法才检查自身配置的低等级知识门槛。
- 知识经验获得后立即进入等级进度并自动升级，不存在可囤积的知识经验池。
- 玩家知识达到功法配置的知识上限后，该功法不再为此知识提供经验。
- 部分实践型知识不能通过修炼增长，只能通过学习、使用或后续生活技能系统增长。

## 2. 功法谱系与功法层

`families` 是玩家看到的完整传承，例如“太虚剑典”。

`methods` 是实际配置与存档使用的独立功法。每个境界层都是一个新功法，通过
`predecessorId` 和 `nextMethodId` 连成升级链。这样可以：

- 独立配置每层知识上限、修炼效率和效果。
- 在升级版中保留核心知识，并加入少量新知识。
- 允许残卷、跳层传承、分支版本和不同品质层独立掉落。

高层功法会递归继承前层：

- 前层知识仍可增长，但每相隔一层，修炼权重乘以 `0.35`。
- 每次升级使继承知识的可修上限提高一级，最高 V 级。
- 当前层再次声明同一知识时，采用更高上限和距离最近的权重。
- 高层不会自动继承旧层效果，只激活当前功法层显式配置的效果。

因此玩家修炼“太虚剑典·化神篇”时，主要领悟剑域，也会继续温养御剑、兵意、剑阵和剑心。

## 3. 修炼与知识增长

每次修炼周期：

```text
功法经验 = 基础修炼值 × 功法效率 × 环境倍率 × 状态倍率
知识经验池 = 功法经验 × knowledgeXpRatio
单项知识经验 = 知识经验池 × normalized(growthWeight)
```

知识只能增长至该功法配置的 `capLevel`。功法熟练度达到 100% 代表完全理解本层功法，
但不代表包含的所有知识均已达到上限。`gainFromCultivation: false` 的知识不会获得修炼经验。

## 4. 知识对功法的反哺

每项知识贡献：

```text
知识掌握率 = min(玩家知识有效等级 / capLevel, 1)
掌握权重 = masteryWeight × capLevel²
综合掌握率 = Σ(知识掌握率 × normalized(掌握权重))
最终效果 = 基础效果 × (1 + masteryBonusMax × 综合掌握率)
```

高等级知识要求具有更高权重，因此掌握核心高阶知识比补齐大量低阶知识更重要。默认
`masteryBonusMax = 0.5`，即相关知识全部达到本功法上限时，功法数值效果提高 50%。

若某效果配置了 `knowledgeScales`，则只使用指定知识计算该效果的额外专精加成。比如剑气伤害
主要受“御剑与剑气”影响，而法力恢复主要受“小周天”影响。

## 5. 功法升级

学习任意功法层通常要求：

- 当前境界达到该层所需境界。
- 普通功法或传承首层不要求知识门槛。
- 特殊功法层可配置一至两项基础知识门槛，最高 II 级。
- 获得该层功法载体、传承或完成领悟事件。

前一层功法与 80% 熟练度只作为推荐进阶路线，不是强制学习条件。高级功法效果基础值应整体
高于低级功法；因知识掌握不足产生的细微下降是允许的。

## 6. 配置约定

最终配置位于 `config/cultivation_methods.json`。

每个 `method` 的主要字段：

| 字段 | 说明 |
|---|---|
| `id` | 稳定唯一 ID |
| `familyId` | 所属玩家感知功法谱系 |
| `tier` | 谱系内层数 |
| `realm` | 本层对应境界 |
| `predecessorId` | 上一层功法 ID，可为空 |
| `knowledge` | 本层知识、等级上限、增长权重、掌握权重及是否可由修炼增长 |
| `effects` | 本层效果、运算方式、叠加组、激活方式和上限 |
| `learningRequirements` | 学习本层所需的独立低等级知识门槛 |
| `recommendedProgression` | 非强制的前层功法与熟练度建议 |

Godot 应将知识实际等级与等级内进度分开存储，以支持连续掌握度：

```text
effectiveLevel = level + currentLevelXp / requiredLevelXp
```

## 7. 平衡护栏

- 同境界混合知识越多，单项知识成长越慢，但功法适应面更广。
- 单知识功法成长快、上限明确，但综合效果较少。
- 高层功法效果不继承旧层；玩家需要根据场景选择当前激活功法。
- 功法效果加成应以知识反哺为主，功法熟练度负责稳定性与解锁，避免双重指数成长。
- 生产、探索类功法和战斗功法使用同一规则，确保非战斗专精拥有长期价值。
- 完整传承之外应持续加入分支、旁门、残卷和无后续层的高专精功法。

## 8. 效果执行规则

每个效果必须明确：

- `operation`：`add_flat` 或 `add_percent`。
- `stackGroup`：同类效果的叠加分组。
- `stackPolicy`：取最高值、封顶相加或唯一生效。
- `activation`：当前版本统一为 `active_method`，只有当前激活功法生效。
- `cap`：百分比效果必须配置硬上限。
- `attributes.category`：修炼、生存、攻击、控制、生产、探索、治理、渡劫或通用。
- `attributes.target`：当前版本主要为 `self`，后续可扩展目标、队友、区域和宗门。
- `attributes.polarity`：正面或负面效果。
- `attributes.valueType`：固定值或比例值。
- `attributes.displayPriority`：Godot 界面中的展示顺序。

`masteryGrowth` 表示功法熟练度从 0% 到 100% 带来的效果增长，不再使用容易混淆的 `perTier`。

每部完整传承的功法层至少配置三个效果：

1. 核心数值效果：体现流派的主要能力。
2. 辅助效果：提供生存、资源或操作补充。
3. 标志性效果：体现该境界层新知识带来的玩法变化。

单知识入门和旁门功法至少配置两个效果。所有效果 ID 在顶层 `effectCatalog` 登记，运行时可据此
映射具体属性、界面文本与结算系统。
