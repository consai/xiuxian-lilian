# 知识树 UI 交互方案

## 1. 系统定位

知识树是正式游戏中的知识掌握与关系浏览界面，不是知识训练界面。

知识没有独立训练队列、训练倒计时或主动训练操作。知识提升来自：

- 修炼包含该知识的功法。
- 研读传承、玉简或完成学习事件。
- 使用相关能力获得实践领悟。
- 后续生活、生产、探索等专业系统。

知识树负责回答：

1. 玩家已经掌握哪些知识，掌握程度是多少。
2. 某项知识影响哪些技能、功法或玩法。
3. 某项知识需要哪些前置。
4. 可以通过哪些实际游戏内容提升该知识。

## 2. 视觉方向

- 严格延续参考图的轻柔暖色绘本风。
- 米黄背景、低对比水彩、圆润深褐线条、灰绿色与浅蓝绿色点缀。
- 使用简单圆角面板和小图标，不使用厚重古卷、复杂雕花或深色木框。
- 保持大量留白，中心知识关系是唯一视觉主体。
- 金色柔光只用于当前选中项，效果与参考图中选中客栈一致。

## 3. 主界面结构

### 顶部

- 左侧：角色名称与当前境界。
- 中部：标题“大道知识”。
- 右侧：按大道、按境界两种浏览方式，以及关闭按钮。
- 不显示训练状态、倒计时或队列入口。

### 左侧大道分类

- 使用简洁圆角按钮展示大道分类。
- 当前大道使用淡金柔光强调。
- 切换大道时保留已标记目标，不改变任何知识状态。

### 中央知识关系图

- 展示当前大道的主要知识关系。
- 跨大道前置从画面边缘接入，并标明所属大道。
- 支持拖拽平移、滚轮缩放和重置视图。
- 单击节点：选中节点并打开详情。
- 双击节点：直接打开“提升途径”弹窗。
- 长按或右键：打开快捷菜单，包含标记目标、查看前置、查看相关内容。

## 4. 节点状态

| 状态 | 表现 |
|---|---|
| 已掌握当前等级 | 灰绿色实心节点 |
| 正在通过某来源成长 | 浅蓝绿色局部填充，不显示时间 |
| 未掌握但条件满足 | 米白节点 |
| 条件不足 | 淡化节点与小型朱砂印 |
| 当前选中 | 墨线稍深，外圈淡金柔光 |
| 已标记目标 | 节点旁显示小旗签 |

节点内只展示名称与当前等级。掌握度、影响和来源放入详情弹窗，避免主树信息过密。

## 5. 附属弹窗

### A. 知识详情

从右侧展开，展示：

1. 知识名称、所属大道与境界。
2. 当前等级和连续掌握度。
3. 知识描述。
4. 对技能、功法与其他系统的影响。
5. 前置知识及当前满足状态。
6. 按钮“查看提升途径”和“查看相关内容”。

### B. 提升途径

居中或从详情面板旁展开，展示该知识当前可用的真实成长来源：

- 功法修炼：列出包含该知识且尚未达到知识上限的功法。
- 研读学习：列出已拥有或可获取的传承、玉简与学习事件。
- 实践领悟：列出可提供知识成长的使用行为或专业系统。

每项来源显示“可用”“尚未获得”“条件不足”或“已达上限”，并提供对应前往操作。
弹窗不包含“开始训练”“加入队列”或训练时间。

### C. 前置与相关内容提示

- 悬停未满足节点时显示轻量提示，列出缺失前置。
- 点击“查看前置”后，高亮相关连接线并弱化无关分支。
- 点击“查看相关内容”后，弹窗列出该知识影响的技能、功法、配方或玩法。
- 玩家可以标记知识为目标，但标记不会自动提升或自动安排内容。

## 6. 核心交互流程

```text
打开知识树
→ 选择“兵道剑道”
→ 点击“剑阵基础 I”
→ 详情显示掌握度、影响与前置
→ 点击“查看提升途径”
→ 查看可用来源：太虚剑典、剑阵初解、剑阵实践
→ 选择“前往功法”或“定位来源”
→ 返回实际游戏系统进行提升
```

## 7. 弹窗与反馈规则

- 知识详情与轻量前置提示可以同时显示。
- “提升途径”和“相关内容”属于聚焦弹窗，打开时收起其他聚焦弹窗。
- 来源可用时使用灰绿色按钮；条件不足时只显示原因，不提供无效按钮。
- 知识等级提升时，节点短暂出现柔和玉色扩散效果。
- 新解锁关系只播放一次连线描绘动画，不持续闪烁。
- `Esc` 优先关闭最上层弹窗，再关闭知识树。
