# 1v5 多波战斗技能设计

## 1. 设计目标

1v5 多波战斗的核心不是让玩家一次清掉所有敌人，而是让玩家在连续压力中做取舍：

- **目标压力**：同屏最多 5 名敌人，后续敌人分波入场，玩家始终处于少打多状态。
- **战斗节奏**：单波 20~35 秒，完整战斗 2~4 波，总时长控制在 90~150 秒。
- **技能判断**：玩家需要在单点击杀、群体压血、防御续航、封技压制、资源回转之间切换。
- **修仙表达**：低阶技能解决眼前敌人，高阶技能改变战场规则，例如剑阵、符阵、领域、法相。

首版技能设计应优先服务自动战斗和配置栏，而不是依赖高频手操。技能效果要能被 AI 规则理解：

- 目标明确：当前可选单体、全体、前排、当前可选低血目标、高威胁目标。
- 触发明确：冷却可用、资源足够、敌人数达到阈值、玩家血量低于阈值。
- 价值明确：伤害、减伤、回复、封技、延后行动、清场、保命。

## 2. 战场结构

### 2.1 站位

默认采用「玩家 1 人 vs 敌方最多 5 槽位」：

| 槽位 | 定位 | 技能交互 |
|---|---|---|
| 前排 1~2 | 近战、护卫、阻挡 | 适合承受单体技能、吃击退和破甲 |
| 中排 1~2 | 输出、控制、特殊行动 | 适合被封技、压制、点杀 |
| 后排 1 | 治疗、召唤、精英 | 默认受保护，适合被标记暴露、穿透、溅射、阵法或领域间接影响 |

技能目标建议保留以下集合，方便后续写入配置：

- `enemy`：当前可选单个敌人。
- `enemies_all`：当前场上全部敌人。
- `enemy_front`：前排敌人。
- `enemy_lowest_hp`：当前可选目标中血量最低敌人。
- `enemy_priority`：当前可选目标中的高威胁敌人。
- `enemy_exposed`：被击退、破阵、标记或前排倒下后暴露的敌人。
- `self`：玩家自身。
- `area`：战场区域效果。

后排不能被主动技能直接指定。只有满足以下条件之一时，后排才会进入可选目标集合：

- 前排敌人全部死亡或被击退。
- 后排单位被标记为高威胁时，仍需要先暴露或通过群攻、穿透、阵法、领域间接处理。
- 后排单位被标记为 `exposed`。
- 技能是群攻、穿透、溅射、阵法或领域效果，按规则间接影响后排。

`enemy` 和 `enemy_lowest_hp` 只从当前可选目标中选择，不会默认越过前排锁定后排。

### 2.2 波次

每场战斗由 2~4 波组成：

| 波次 | 敌人构成 | 设计目的 |
|---|---|---|
| 第 1 波 | 2~3 个普通敌人 | 让玩家建立资源与冷却节奏 |
| 第 2 波 | 3~5 个普通敌人，混入治疗、召唤或控制敌人 | 检查群攻、封技压制和续航能力 |
| 第 3 波 | 精英 + 护卫，或 5 个高压敌人 | 检查爆发窗口与保命技能 |
| 第 4 波 | 可选首领波 | 用于秘境、剧情、Boss 事件 |

波次切换时不应完全重置玩家状态。建议保留：

- 当前气血。
- 当前法力/体力。
- 技能冷却。
- 已开启的持续技能。
- 战斗内 Buff/Debuff，按剩余时间自然衰减。

这样多波战斗才会形成「上一波的选择影响下一波」。

## 3. 技能分类

### 3.1 战斗主动

主动技能负责解决眼前问题。首版至少需要 6 类：

| 类别 | 作用 | 典型触发 |
|---|---|---|
| 单点斩杀 | 快速减少敌人数 | 敌方低血、精英暴露 |
| 群体压血 | 对抗 4~5 敌同场 | 敌人数 >= 3 |
| 封技压制 | 推迟治疗、召唤、大招 | 当前可选目标中存在高威胁敌人 |
| 防御保命 | 抵御集火 | 玩家血量 <= 40% |
| 回复回转 | 延长多波续航 | 玩家血量或法力低 |
| 位移/突袭 | 制造暴露窗口或摆脱集火 | 后排存在高威胁敌人，但仍受前排保护 |

### 3.2 战斗被动

被动技能负责定义流派倾向，而不是提供必须手动判断的瞬时效果：

- 剑修：击杀后刷新部分冷却，鼓励连斩。
- 法修：敌人数越多，法术效率越高。
- 体修：受击叠加韧性，越打越稳。
- 符修：首次命中敌人时附加标记，后续技能引爆。
- 阵修：每波开始铺设阵势，越长线越强。

### 3.3 持续技能

持续技能适合表现修仙战斗的「展开状态」：

- 护体真气：持续耗蓝，获得减伤。
- 剑阵：持续耗蓝，对全体敌人周期伤害。
- 五行领域：持续耗蓝，降低敌方行动速度并提升自身法术伤害。
- 法相显化：持续耗体力和法力，大幅提升生存与近战伤害。

持续技能应有明显开关条件，避免自动战斗乱开：

- 敌人数 >= 3 时开启群体领域。
- 玩家血量 <= 50% 时开启护体。
- 精英或首领出现时开启法相。
- 法力低于 20% 时关闭非保命持续技能。

## 4. 1v5 技能设计原则

### 4.1 避免纯单体支配

如果所有强技能都是单体，多波 1v5 会变成单纯数值碾压。首版建议：

- 单体技能倍率最高，但冷却较长。
- 群体技能总伤害高，单个目标伤害低。
- 控制技能伤害低，但能封技、减速或延后敌方关键行为。
- 防御技能不直接清怪，但能让玩家撑过敌人齐射窗口。

### 4.2 用敌人数驱动技能价值

同一个技能在不同敌人数下应有不同价值：

| 敌人数 | 推荐技能价值 |
|---|---|
| 1 个 | 单点爆发、破甲、斩杀 |
| 2~3 个 | 小范围伤害、点控、续航 |
| 4~5 个 | 群攻、领域、护盾、群体减速 |

### 4.3 让封技成为关键按钮

多波战斗里，敌人治疗、召唤、强力行动会制造压力峰值。由于释放技能时战斗暂停，不设计实时施法目标；控制技能改为影响敌人的下一次特殊行动：

- 低阶：使目标下一次特殊行动延后。
- 中阶：封印目标技能 1 次，只允许其进行普通攻击。
- 高阶：领域内敌人行动速度降低，特殊行动冷却增加。

### 4.4 续航比爆发更重要

多波战斗要避免「第一波全交技能，后面只能挨打」。技能资源建议分层：

- 法力：法术、领域、护盾。
- 体力：体修、身法、近战爆发。
- 冷却：控制高价值技能频率。
- 击杀/破势触发：奖励正确集火与收割。

## 5. 首版技能池

以下技能用于建立首版 1v5 多波体验。数值是设计基准，实际落表时可按境界倍率和敌人曲线调整。

### 5.1 练气期

| 技能 | 类型 | 定位 | 目标 | 建议效果 |
|---|---|---|---|---|
| 御气弹 | 主动 | 基础单体 | `enemy` | 低消耗单体灵力伤害，作为默认填充技能 |
| 流风步 | 主动 | 保命/位移 | `self` | 获得短暂闪避窗口；若目标已暴露，下一次单体技能优先命中暴露目标 |
| 火鸦术 | 主动 | 小群攻 | `enemies_all` | 对最多 3 个敌人造成火属性伤害 |
| 凝气护身 | 主动 | 防御 | `self` | 获得 5 秒护盾，敌人数越多护盾越高 |
| 灵息回转 | 主动 | 资源 | `self` | 回复少量法力；若当前为第 2 波以后，额外回复 |

练气期目标：让玩家理解「不能只带伤害技能」，至少需要一个群攻或防御。

### 5.2 筑基期

| 技能 | 类型 | 定位 | 目标 | 建议效果 |
|---|---|---|---|---|
| 破空剑气 | 主动 | 单点爆发 | `enemy_lowest_hp` | 高倍率剑伤，若击杀则返还部分冷却 |
| 五行轮爆 | 主动 | 群体压血 | `enemies_all` | 对所有敌人造成五行伤害，敌人数越多总收益越高 |
| 震血拳 | 主动 | 控制/破势 | `enemy_front` | 物理伤害并削弱目标攻击，适合打前排 |
| 缚灵符 | 主动 | 封技 | `enemy_priority` | 对当前可选高威胁目标造成少量伤害，并使其下一次特殊行动延后 |
| 护体真气 | 持续 | 长线防御 | `self` | 持续耗蓝，获得减伤和少量抗性 |
| 剑心通明 | 被动 | 收割流 | `self` | 击杀敌人后回复少量法力并缩短剑系技能冷却 |

筑基期目标：形成第一批流派分化，剑修收割、法修群攻、符修控制、体修抗压。

### 5.3 金丹期

| 技能 | 类型 | 定位 | 目标 | 建议效果 |
|---|---|---|---|---|
| 金丹真火 | 主动 | 精英压制 | `enemy` | 高额火伤，目标有护盾时额外破盾 |
| 雷音震魄 | 主动 | 群体压制 | `enemies_all` | 中等雷伤，使敌方下一次特殊行动整体延后 |
| 三才剑阵 | 持续 | 阵地群伤 | `area` | 每 2 秒对全体敌人造成剑阵伤害 |
| 玄元归息 | 主动 | 波间续航 | `self` | 回复气血；若刚进入新波次，效果提升 |
| 丹火护脉 | 被动 | 防爆发 | `self` | 每波首次低血时自动获得护盾和短暂减伤 |

金丹期目标：开始让玩家能应对 4~5 同场和连续精英压力。

### 5.4 元婴及以后

| 技能 | 类型 | 定位 | 目标 | 建议效果 |
|---|---|---|---|---|
| 元婴出窍 | 主动 | 双线输出 | `enemy_exposed` | 分身锁定暴露目标；若无暴露目标，则协同攻击当前目标并尝试施加暴露 |
| 小五行领域 | 持续 | 战场规则 | `area` | 敌方行动速度降低，自身五行技能伤害提升 |
| 万剑归宗 | 主动 | 清场爆发 | `enemies_all` | 对全体造成多段剑伤，敌人越少单体伤害越高 |
| 法相显化 | 持续 | 终局形态 | `self` | 提升气血上限、减伤和近战伤害，持续消耗资源 |
| 因果锁命 | 主动 | 首领/精英 | `enemy` | 标记目标，期间其受到的部分伤害在标记结束后结算 |

高阶目标：让技能从「解决一个敌人」升级为「改写战斗规则」。

## 6. 推荐初始配置

首版玩家默认 5 个技能槽，可按下面给出一套通用组合：

| 槽位 | 技能 | 用途 |
|---|---|---|
| 1 | 御气弹 | 填充输出 |
| 2 | 火鸦术 / 五行轮爆 | 群体压血 |
| 3 | 缚灵符 | 封技压制高威胁目标 |
| 4 | 凝气护身 / 护体真气 | 防御 |
| 5 | 破空剑气 | 收割低血目标 |

自动战斗预设建议：

- **均衡**：封技 > 低血收割 > 群攻 > 防御 > 填充。
- **进攻**：低血收割 > 群攻 > 单体高伤 > 封技 > 填充。
- **保守**：防御/回复 > 封技 > 群攻 > 收割 > 填充。

## 7. 敌人与技能反制关系

为了让技能选择有意义，敌人也需要明确职责：

| 敌人类型 | 威胁 | 推荐玩家解法 |
|---|---|---|
| 护卫 | 保护后排、削弱单体爆发 | 破甲、群攻、前排控制 |
| 控制者 | 周期性群控或减益 | 封技、压制、制造暴露后点杀 |
| 治疗者 | 拉长战斗，消耗玩家资源 | 清前排、标记暴露、群攻或穿透 |
| 召唤者 | 增加敌人数，制造 1v5 压力 | 封技压制或快速清场 |
| 精英 | 高血高伤，常带阶段技能 | 爆发窗口、防御循环、领域 |

每波至少应出现一个能被技能明显克制的敌人，否则技能选择会退化为纯 DPS 对比。

## 8. 配置落地建议

### 8.1 `jineng.yaml`

已有技能配置可以继续使用当前结构。为了支持 1v5 多波，建议后续新增或约定以下字段语义：

- `combat.target` 支持 `enemies_all`、`enemy_front`、`enemy_lowest_hp`、`enemy_priority`、`enemy_exposed`。
- `effects[].target` 与 `combat.target` 保持一致，必要时允许单个技能有多个效果目标。
- `tags` 增加 `area`、`seal`、`defense`、`execute`、`wave_sustain`、`upkeep`。
- `trigger` 用于自动战斗建议，不直接替代 AI 规则。

示例：

```yaml
trigger:
  recommendedWhen:
    enemyCountGte: 3
    playerHpLte: null
    hasPriorityEnemy: false
```

### 8.2 `xiaoguo_catalog.yaml`

首版优先复用已有效果 ID。若需要补充，建议集中加入：

| effectId | 用途 |
|---|---|
| `delay_special_action` | 延后下一次特殊行动 |
| `seal_special_action` | 封印下一次特殊行动 |
| `shield_value` | 护盾值 |
| `wave_start_bonus` | 新波次触发加成 |
| `cooldown_refund_on_kill` | 击杀返还冷却 |
| `mana_restore_on_kill` | 击杀回蓝 |
| `enemy_count_scaling` | 按敌人数放大效果 |

### 8.3 战斗运行时

运行时建议把波次状态放在 `DataStore.zhandou_runtime()` 或战斗初始化信封中，不在技能脚本里保存跨场景状态：

- `wave_index`
- `wave_count`
- `enemies_alive`
- `enemies_spawned_this_wave`
- `battle_elapsed`
- `wave_elapsed`

技能本身只读取战斗上下文，不持久保存进度。

## 9. 数值基准

以下基准用于首版调参：

| 项目 | 建议值 |
|---|---|
| 单体填充技能 CD | 2.5~4 秒 |
| 单体爆发技能 CD | 5~8 秒 |
| 群体技能 CD | 8~14 秒 |
| 封技技能 CD | 10~16 秒 |
| 防御技能持续 | 4~8 秒 |
| 持续技能消耗 | 每秒 3%~6% 基础法力恢复量 |
| 单波目标时长 | 20~35 秒 |
| 完整战斗目标时长 | 90~150 秒 |

伤害预算建议：

- 单体填充：1.0 倍基准伤害。
- 单体爆发：1.6~2.2 倍基准伤害。
- 群体技能：每个目标 0.45~0.75 倍基准伤害。
- 封技技能：0.4~0.8 倍基准伤害，价值主要来自控制。
- 防御技能：抵消约 1.5~2.5 次普通攻击。

## 10. 首版验收标准

技能系统接入 1v5 多波后，至少满足以下体验：

- 玩家只带单体技能时，可以赢低难度，但在 4~5 敌同场时明显吃力。
- 玩家带 1 个群攻技能后，第 2 波压力明显下降。
- 玩家带 1 个封技技能后，治疗者/召唤者/控制者波次明显更稳定。
- 玩家带 1 个防御或回复技能后，多波续航明显改善。
- 自动战斗三种预设在同一套技能下有不同表现。
- 高阶持续技能能改变整场战斗节奏，但会受到资源限制。

## 11. 后续扩展

后续可以沿三条线扩展：

- **流派线**：剑修收割、法修清场、体修抗压、符修控制、阵修长线、丹修续航。
- **敌人线**：护卫、治疗、召唤、控制、精英、首领阶段。
- **波次线**：伏击波、增援波、精英波、Boss 波、限时生存波。

首版不需要一次做全。只要先做出「群攻有用、封技有用、防御有用、收割有用」四件事，1v5 多波战斗的技能骨架就成立了。
