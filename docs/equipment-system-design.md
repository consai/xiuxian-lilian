## 设计目标

装备系统的核心目标不是让玩家多穿几件加数值，而是提供构筑微调：

1. **主动选择保持稀少**：战斗中只保留 2 个主动法宝位，避免自动战斗和手动操作都变臃肿。
2. **常驻装备补短板**：被动法宝承担属性、抗性、续航和流派词条。
3. **探索法宝连接战斗外收益**：移动、储物、采集、避险等法宝影响历练决策。
4. **本命法宝做长期投入**：后期提供身份感和成长线，但首版不实现。

|---|---:|---|---|---|

| 护身法宝 | 1 | 炼气中期 `[PLACEHOLDER]` | 常驻防御、抗性、护盾增强 | 补生存还是补资源 |
| 身法法宝 | 1 | 筑基 `[PLACEHOLDER]` | 追逃、旅行安全 | 跑图收益还是战斗机动 |
| 储物法宝 | 1 | 筑基后 `[PLACEHOLDER]` | 背包容量、采集、带回奖励 | 多拿材料还是提升战斗 |
| 本命法宝 | 1 | 金丹 `[PLACEHOLDER]` | 长线养成、绑定流派 | 专精路线承诺 |

首版实现只建议做前三类：主动法宝、护身法宝、身法法宝。储物和本命先留设计位。

## 装备分类

| 类型 | 对应现有数据 | 是否进战斗按钮 | 允许效果 |
|---|---|---:|---|
| 主动法宝 | `主动法宝` / `data/zhuangbei.yaml` | 是 | 伤害、护盾、治疗、控制、回蓝 |
| 被动法宝 | `被动法宝` | 否 | 固定属性、百分比属性、抗性、触发词条 |
| 移动法宝 | `移动法宝` | 否 | 身法、追逃、旅行时间、历练避险 |
| 储物法宝 | `移动法宝` 或新增 `储物法宝` | 否 | 背包容量、带回上限、采集加成 |
| 本命法宝 | 后续新增 | 可附带 1 个主动 | 成长词条、流派机制、重铸代价 |

若新增离散类型，实现时应加到 `scripts/enum/`，不要在业务脚本散写魔法字符串。

## 核心机制

### 装备 / 卸下

**目的**：让玩家在出发前调整构筑。  
**玩家体验目标**：换装成本低，试错不肉疼。  
**输入**：玩家点击槽位，选择背包或已拥有法宝。  
**输出**：写入 `DataStore.savedata` 中的装备槽位，刷新属性和战斗快照。  
**失败状态**：

- 未拥有该装备：提示“未拥有该法宝”。
- 类型不匹配：提示“该法宝不能装备到此槽位”。
- 同一法宝重复装备：自动从旧槽位移除。

首版普通装备卸下不收费。本命法宝重铸才需要代价。

### 主动法宝

**目的**：提供战斗中的高影响力决策。  
**玩家体验目标**：2 个按钮足够形成取舍，不把战斗变成技能栏管理。  
**输入**：战斗中手动释放或自动战斗策略触发。  
**输出**：造成效果并进入冷却。  
**边界**：

- 主动法宝数量固定 2 个，不随境界继续增加。
- 主动法宝强度按“同境界循环技能的 1.5–3.0 倍效果预算”估算 `[PLACEHOLDER]`。
- 冷却建议 `6–18` 秒 `[PLACEHOLDER]`，越偏控制/保命越长。

### 常驻法宝

**目的**：给构筑提供稳定补强。  
**玩家体验目标**：玩家能用装备修正短板，而不是无脑堆最高战力。  
**输入**：装备在护身、身法、储物等常驻槽。  
**输出**：进入统一属性加值池或历练规则修正。  
**边界**：

- 同阶同品质单件常驻法宝预算不超过主功法常驻预算的 `20–35%` `[PLACEHOLDER]`。
- 百分比词条进入统一加值池，不新增独立乘区。
- 一件装备最多有 1 个核心词条 + 1 个副词条，首版不要随机词条。

### 强化

**目的**：提供装备消耗口。  
**玩家体验目标**：喜欢某件法宝时可以继续投入，但不会被旧装备绑死。  
**输入**：灵石、材料、同类胚子 `[PLACEHOLDER]`。  
**输出**：提升基础值或冷却/效果小幅改善。  
**失败状态**：材料不足、阶位上限不足、装备被锁定。

首版可跳过强化，只做获得与装备；等掉落、商店和材料循环稳定后再加。

## 数值原则

| 项 | 首版假设 | 理由 |
|---|---:|---|
| 主动法宝位 | 2 | 已实现，足够产生战斗策略取舍 |
| 常驻法宝位 | 2–3 `[PLACEHOLDER]` | 补构筑，不喧宾夺主 |
| 主动法宝冷却 | 6–18 秒 `[PLACEHOLDER]` | 覆盖短循环、防守窗和爆发窗 |
| 单件常驻预算 | 主功法的 20–35% `[PLACEHOLDER]` | 装备不能取代功法 |
| 同阶品质差 | 约 8–15% `[PLACEHOLDER]` | 品质有感，但阶位仍更重要 |
| 卸装成本 | 0 | 鼓励试错 |
| 本命重铸成本 | 高 `[PLACEHOLDER]` | 后期承诺感来源 |

所有 `[PLACEHOLDER]` 值都需要战斗模拟和历练收益测试后定稿。

## UI 流程

1. 战斗配置页保留当前“主动法宝”区域，显示 2 个主动槽。
2. 角色面板新增“装备”页，展示护身、身法、储物、本命等常驻槽。
3. 点击槽位打开背包筛选，只显示匹配类型。
4. 槽位上显示：图标、名称、阶位、品质、核心效果短句。
5. 详情弹窗显示：基础属性、主动效果、冷却、限制、来源。

固定 UI 节点应写在 `.tscn` 中；脚本只负责绑定数据和处理信号。

## 数据建议

后续实现时，不急着替换旧字段。先兼容：

```gdscript
equipment_slots = {
	"active": [-1, -1],
	"guard": -1,
	"movement": -1,
	"storage": -1,
	"life_bound": -1,
}
```

迁移规则：

- 旧 `equip_slots` / `treasure_item_slots` 继续视为 `active`。
- 新常驻槽写入 `equipment_slots`。
- 战斗快照只读取 `active` 生成按钮；常驻槽通过属性构筑服务进入 `attrs`。

## 实现顺序

1. **P0：类型与文案**  
   统一主动/被动/移动法宝分类，背包筛选按类型工作。

2. **P1：护身与身法槽**  
   增加 `equipment_slots.guard`、`equipment_slots.movement`，只做固定属性加成。

3. **P2：角色装备页**  
   新建可复用槽位场景，角色面板组合使用。

4. **P3：探索与储物**  
   将移动、储物法宝接入历练速度、逃跑、带回上限。

5. **P4：本命法宝**  
   加绑定、成长、重铸与流派词条。

## 验收标准

- 玩家不进战斗也能看懂每个槽位的职责。
- 主动法宝仍只有 2 个按钮。
- 常驻法宝能改变构筑短板，但不会让功法和技能失去意义。
- 同一件装备不能重复占多个槽。
- 新存档有默认槽位，旧存档读档后不丢当前 2 个主动法宝。
