# 人物基础属性与等级曲线基准

> 目标：所有玩家、怪物、精英、Boss 的首版面板都以本文为基准生成。先定“同等级标准玩家”，再按遭遇强度带和怪物模板换算敌人。

配置落点：玩家每级基础四维与基础战斗面板在 `data/exportjson/realms.json`，怪物基准在 `data/exportjson/yunxing_params/jingjie_balance_benchmark_ene.json`。

---

## 1. 等级与大境界

人物使用 `Lv1-Lv90` 的内容等级。大境界从 `Lv10` 开始每 10 级发生一次质变：

| 等级 | 大境界 | 设计定位 |
|---:|---|---|
| 1-9 | 炼气 | 学会基础资源循环，建立主属性方向 |
| 10-19 | 筑基 | 第一次质变，解锁稳定构筑入口 |
| 20-29 | 金丹 | 核心机制成型，技能爆发窗口变明确 |
| 30-39 | 元婴 | 分身、替身、强控制或复苏类机制开始出现 |
| 40-49 | 化神 | 规则型能力出现，单纯数值不再决定胜负 |
| 50-59 | 炼虚 | 空间、阵域、远程操控成为常规内容 |
| 60-69 | 合体 | 多系统构筑合流，Boss 机制强度提高 |
| 70-79 | 大乘 | 接近终局，装备、功法、道法组合要求很高 |
| 80-90 | 渡劫 | 终局压力，天劫、因果、法则抗性成为主题 |

当前境界与模拟参数由 `data/exportjson/realms.json` 和 `data/exportjson/yunxing_params/moni*.json` 导出；怪物与战斗面板制作时以本文的 `content level` 为主。

## 2. 基础属性

继续沿用四维根基：

| 属性 | 键 | 战斗身份 | 非战斗身份 |
|---|---|---|---|
| 肉身 | `roushen` | 气血、物攻、物防、控制抗性 | 负重、采集、环境承载 |
| 灵力 | `lingli` | 法力、法攻、法防、控制强度 | 阵法驱动、飞行续航 |
| 神识 | `shenshi` | 控制、反应、法攻补正 | 感知、炼丹炼器精度 |
| 身法 | `shenfa` | 追逃、行动速度 | 探索机动、旅行安全 |

资质属性不直接堆战斗面板：

| 资质 | 键 | 设计职责 |
|---|---|---|
| 悟性 | `comprehension` | 功法、知识、技能领悟效率 |
| 心性 | `will` | 心魔、渡劫、长期修持稳定性 |
| 福缘 | `fortune` | 稀有事件权重、保底推进、奇遇质量 |
| 灵根 | `roots` | 元素适配、修炼路线、功法条件 |

## 3. 标准玩家根基公式

同等级标准玩家的四维均值：

```text
大境界序号 M = 0..8（Lv1-9 为 0，Lv10-19 为 1，以此类推）
标准四维均值 foundation_avg = 10 + 0.8 * (level - 1) + 4.0 * M
```

其中：

- `10` 是 Lv1 炼气初始四维。
- `0.8` 是每级每项平均成长，前 10 级每次升级都要看得见面板变化。
- `4.0 * M` 是大境界脉冲，约等于 5 个普通等级的根基成长，保证 Lv10、Lv20、Lv30 等节点有质变。

实际升级奖励建议拆为：

```text
每级根基点 = 4
每点根基 = +0.6 指定基础属性
等级共鸣 = 四维各 +0.2
标准均衡角色 = 四维各约 +0.8 / 级
大境界突破脉冲 = 四维各 +4.0
```

流派角色可偏离均值，但总预算不变：

| 构筑 | 主属性 | 副属性 | 牺牲属性 |
|---|---:|---:|---:|
| 均衡 | `foundation_avg` | `foundation_avg` | `foundation_avg` |
| 专精 | `foundation_avg * 1.15` | `foundation_avg * 1.05` | `foundation_avg * 0.90` |
| 极限 | `foundation_avg * 1.25` | `foundation_avg` | `foundation_avg * 0.82` |

极限构筑只用于玩家或少量标志敌人；普通怪物不使用极限模板。

## 4. 战斗面板公式

进入战斗时先取当前 `realms.json` 行的基础面板，再叠加功法、装备等运行时修正：

```text
基础面板 = realms.json[当前等级].combat_attrs
最终面板 = (基础面板 + 其他固定值) * (1 + 同类百分比之和)
```

核心推导公式来自 `data/exportjson/yunxing_params/jingjie_balance_combat_attrib.json`：

| 面板 | 公式 |
|---|---|
| 气血 | `roushen * 10 + 80` |
| 法力 | `lingli * 5 + 80` |
| 物攻 | `roushen * 2.5` |
| 法攻 | `lingli * 2.3 + shenshi * 0.5` |
| 物防 | `roushen * 1.2` |
| 法防 | `lingli * 1.1 + shenshi * 0.8` |
| 速度 | `shenfa * 2 + shenshi * 0.5 + 100` |
| 控制强度 | `shenshi * 1.6 + lingli * 0.5` |
| 控制抗性 | `roushen + shenshi * 1.2` |

## 6. 大境界质变规则

大境界不是单纯“多给数值”，每次突破必须至少带来一个规则变化：

| 质变项 | 建议实现 |
|---|---|
| 面板跃迁 | 标准战力相对前一等级提高约 `18%-30%`，其中 Lv9→10、Lv19→20 等节点必须明显高于普通升级 |
| 构筑扩展 | 新功法槽、技能位、专属资源或被动槽至少解锁一类 |
| 内容换层 | 新区域、新敌人模板、新材料阶位开启 |
| 风险变化 | 新负面机制出现，如心魔、丹毒、天劫、法则压制 |
| 反制方式 | 同步给玩家至少一种新解法，不只提高怪物数值 |

突破品质可以影响根基成长和少量永久加成，但一品与下品的长期战斗面板差距应控制在 `8%-12%` 内，避免滚雪球。

## 7. 怪物设计换算

以后新增怪物按以下顺序制作：

1. 确定怪物所在内容等级 `L`。
2. 用本文公式生成同等级标准玩家面板。
3. 选择遭遇强度带：弱敌、普通、强敌、精英、Boss。
4. 选择怪物模板：猛攻、坚守、游斗、控制、召唤等。
5. 用“强度带系数 * 模板系数”换算面板。
6. 精英和 Boss 再补机制，不用纯面板硬堆。

强度带继续沿用 `encounter_bands`：

| 类型 | 总强度预算 |
|---|---:|
| 弱敌 | 55-70 |
| 普通 | 80-95 |
| 强敌 | 105-120 |
| 精英 | 140-170 |
| Boss | 220-300 |

模板系数记录在 `monster_design_baseline.role_templates`：

| 模板 | 气血 | 攻击 | 防御 | 速度 | 说明 |
|---|---:|---:|---:|---:|---|
| 猛攻 `striker` | 0.85 | 1.18 | 0.85 | 1.05 | 爆发高，有明显击杀窗口 |
| 坚守 `guardian` | 1.25 | 0.90 | 1.20 | 0.85 | 耐打，怕破防或持续伤害 |
| 游斗 `skirmisher` | 0.75 | 0.95 | 0.80 | 1.25 | 快速行动，怕必中和范围 |
| 控制 `controller` | 0.90 | 0.80 | 0.95 | 1.00 | 控制强，正面输出低 |
| 召唤 `summoner` | 0.80 | 0.75 | 0.85 | 0.95 | 本体弱，强度在召唤机制 |

Boss 纯属性预算最多占总强度的 `60%`，剩余强度必须来自可观察、可学习、可反制的机制。

## 8. 验收标准

每次新增等级段或批量怪物后，至少验证：

| 指标 | 合格范围 |
|---|---|
| 同等级标准玩家打普通怪 | 胜率 `70%-85%` |
| 同等级成型玩家打普通怪 | 胜率 `90%-98%` |
| 同等级标准玩家打精英 | 胜率 `35%-55%` |
| 低一大境界成型玩家打高一境界普通怪 | 可挑战，但胜率低于 `50%` |
| 单一属性极限堆叠相对混合构筑收益 | 不超过 `20%` |
| Boss 纯面板强度占比 | 不超过 `60%` |

数值调整时先改标准玩家母线，再改怪物模板；不要为单个怪物临时发明一套新标尺。
