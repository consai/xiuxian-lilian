# 战斗规则

本文档定义当前战斗系统的产品规则。实现、UI 文案和测试应以此为准。

## 1. 战斗模式

战斗采用 **Wait-ATB（等待式行动条）**：

- `ADVANCING`：战斗模拟时间推进。
- `PAUSED`：一方行动条满，等待该方选择并结算行动；模拟时间冻结。
- `PRESENTATION`：播放行动表现；模拟时间冻结。
- `END`：战斗结束，不再推进。

玩家手动操作时拥有无限决策时间。这是当前模式的明确规则，不是实时 ATB。

## 2. 统一模拟时钟

只有 `ADVANCING` 消耗战斗模拟时间。以下系统使用同一模拟时钟：

- 玩家和敌人的行动条
- 技能、法宝和战斗物品冷却
- Buff 与 Debuff 剩余时间
- DoT、HoT 等周期效果
- 战斗推进时限

`PAUSED` 和 `PRESENTATION` 期间，上述系统全部冻结。

UI 中显示的秒数均指战斗模拟秒，不代表现实经过时间。

## 3. 行动条

- 行动条周期由速度属性决定。
- 双方行动条仅在 `ADVANCING` 推进。
- 任意一方行动条满时，战斗进入 `PAUSED`。
- 双方在同一次推进中同时满条时，玩家优先获得行动权。
- 行动结算与表现完成后，出手方行动条重置；未出手方保留已有进度。
- 单帧超过行动条上限的出手方溢出进度会被保留，最多保留一个行动条周期。

## 4. 行动与表现

- 数据结算发生在 `PAUSED`。
- 数据结算完成后进入 `PRESENTATION`。
- 表现完成后检查死亡，并恢复 `ADVANCING`。
- 表现时长不得影响冷却、Buff、DoT、行动条或战斗推进时限。

## 5. Buff、DoT 与冷却

- Buff、Debuff、DoT 和所有冷却只在 `ADVANCING` 推进。
- 周期伤害可以在推进阶段造成死亡，并立即结束战斗。
- 玩家思考期间，双方的正面与负面状态均冻结。

## 6. 战斗推进时限

默认时限为 200 个战斗模拟秒。

- 时限只累计 `ADVANCING` 时间。
- 玩家决策和表现播放不消耗时限。
- 达到时限时战斗结束，当前规则判定玩家失败。

代码中的 `battle_elapsed_advancing` 是该时限的权威计时值。产品文案应使用“战斗推进时限”，避免称为现实意义上的“整场时限”。

## 7. 规则变更要求

若未来切换为实时 ATB，必须先更新本文档，并重新定义：

- 玩家选择技能时敌方能否行动
- 多个行动同时就绪时的排队规则
- 选择期间角色死亡的处理
- 表现期间是否继续模拟
- 现实时间与模拟时间的关系

任何规则变更都应同步更新无界面战斗域测试。
