## 当前排队

### [ ] SF-01：结算提示按退出原因区分

位置：`scripts/lilian/lilian_jiesuan.gd`

现象：结算底部提示已经能给恢复方向，但战败、撤退、主动返程可以更明确。

验收：战败提示恢复，主动返程提示整理收获，撤退提示轻成本和再准备。

### [ ] SF-02：洞府上次历练消息补恢复建议

位置：`scripts/sim/dongfu.gd`

现象：玩家战败回洞府后，不一定知道先休息、炼丹、研读或调整技能。

验收：上次历练失败时，洞府消息明确给出一条下一步建议。

### [ ] SF-03：典籍物品提示补研读入口

位置：`data/exportjson/item_items.json`、`scripts/ui/item_info_payload_builder.gd`

现象：玩家觉得没有新功法 / 技能学习，实际问题是学习入口不够明显。

验收：典籍类物品提示能指向洞府底部「研读」入口。

### [ ] SF-04：疗伤丹描述与效果一致

位置：`data/exportjson/item_items.json`

现象：疗伤丹如果只恢复气血，描述不能暗示会治疗伤势。

验收：物品描述与实际效果一致；若改成治疗伤势，必须补 `run_simulation_tests.gd` 覆盖。
