# P0 手动验收清单

## 准备

- 确认当前分支无需要保留的临时测试目录影响结果。
- 优先运行 `.\tests\run_all_tests.ps1`；若只改历练 / 导航，至少运行对应单测。
- Godot 主场景从 `scenes/ui/main_menu.tscn` 启动。

## 20 分钟游玩路径

1. 新建游戏，进入洞府。
2. 打开角色属性、背包、功法 / 技能配置面板，确认能返回洞府。
3. 从洞府进入世界地图，选择可用地点开始历练。
4. 在历练地图推进到事件节点，确认日志更新。
5. 遇到战斗后进入战斗浮层，确认底层历练界面被隐藏且状态未丢。
6. 战斗胜利后返回原历练界面，继续推进至少一个节点。
7. 历练中打开背包，使用一枚可用丹药，关闭后回到原历练界面。
8. 主动结算或战败结算，确认奖励、消耗、耗时和日志显示。
9. 返回洞府，确认背包奖励、角色状态、活动日志和自动存档结果。

## 必查结果

- 历练中不能读档 / 手动存档。
- 战斗中不能通过背包绕过丹药使用限制。
- 历练胜利返回不重新生成地图。
- 战败 / 撤退只结算一次。
- 结算后 `GameState.last_expedition_summary` 有摘要，`ExpeditionState.active` 为 false。
- 回洞府后可再次进入世界地图。
