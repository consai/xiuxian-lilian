# 项目路线图

## P0：可玩闭环稳定版

状态：已完成。`tests/run_all_tests.ps1` 已通过。

目标：玩家能从新局进入洞府，完成一次历练、一次战斗、一次结算，并把奖励带回成长系统。

范围：

- 历练中战斗浮层、背包浮层、结算返回链路稳定。
- 背包、技能配置、角色面板在历练内外上下文清晰。
- 战斗胜败、撤退、结算奖励、自动存档不破坏运行时状态。
- 战斗特效只做最低可读版：命中、受击、治疗、胜败反馈清楚即可。
- 配置校验和现有 headless 测试全绿。

验收：

- `tests/run_all_tests.ps1` 通过。
- 手动流程：新局 -> 洞府 -> 世界地图 -> 青岚山 / 可用地点 -> 历练 -> 战斗 -> 结算 -> 回洞府。
- 历练中不能误存档 / 误读档；结算后可正常自动存档。

## P1：新手体验与引导版

状态：已完成。看板验收已记录。

目标：把已有教程、剧情播放和功能入口串成首章体验。

范围：

- 洞府设定调整为山顶清修地，并同步主界面背景、入口文案和新手叙事口径。
- 新手剧情步骤和洞府 / 历练 / 第一场战斗事件对齐。
- 引导遮罩只引导必要点击，不强制做完整教学百科。
- 第一次奖励、第一次学习技能 / 功法、第一次炼丹入口给出明确反馈。
- 补第一批稍正式人物表现：主角默认形象、引导相关角色、首战敌人头像 / 立绘占位替换。

验收：

- 老存档不被强制拉回新手流程。
- 新局能按 `docs/new-player-guidance-story.md` 的 T00-T10 完成。
- 跳过教程后核心入口仍可用。

## P2：数值与内容校准版

状态：已完成。看板验收已记录。

目标：练气到筑基前的成长、资源、战斗难度可测。

范围：

- `pingheng_v1.yaml`、境界、玩家基础属性、怪物、奖励曲线统一校准。
- 炼丹、突破、历练奖励形成短期目标。
- 功法 / 技能配置按首版池收敛，暂不扩高阶境界。
- 战斗特效升级到体验版：按技能类型区分弹道、受击、状态、浮字节奏。

验收：

- 30-60 分钟内能看到明确成长目标。
- 同境界普通战斗不秒杀、不拖沓。
- 筑基前主要资源不会出现必然卡死或无限膨胀。

## P3：内容扩展版

状态：已完成。看板验收已记录。

目标：在稳定闭环上增加地图、事件、技能流派和长期目标。

范围：

- 增加 1 个新区域、1 条事件链和一组怪物组合。
- 扩展练气后段到筑基初期的功法、技能和大道知识节点。
- 增加 2-3 个炼丹配方、突破辅助道具和地图节点状态。
- 扩正式人物资产：主要 NPC、区域敌人、流派代表形象。

验收：

- 每批新增内容都有配置校验。
- 新内容不要求新架构，除非现有服务无法表达。
- 每批内容至少补一条可重复的手动验收路径。
- 不启动高阶境界完整投放；只做到筑基初期可继续玩。

## P4：体验修正与打磨版

状态：当前阶段。

目标：解决实际游玩中“打不过、目标不清、操作别扭、反馈不舒服”的问题，先修体验，不扩新系统。

范围：

- 根据手动游玩反馈调整早期战斗难度、恢复资源、失败成本和奖励节奏。
- 优化关键流程的小卡点：入口文案、按钮反馈、结算说明、突破目标提示。
- 修正 P3 新区域导致的难度跳跃或资源断档。
- 只做能改善当前体验的小优化；新区域、新境界、新机制放后续版本。

验收：

- 新局到筑基前目标路线不被普通战连续卡死。
- 玩家至少能通过准备丹药、调整技能或换地点找到解法。
- 失败后仍有可执行的恢复路径。
- 小优化不破坏 P0-P3 自动回归。

## P5：配置扩容版

状态：待 P4 体验修正通过后进入。

目标：在手感稳定后补药材、丹药、怪物、掉落和区域配置，扩内容量，不改核心系统。

范围：

- 增加药材族、材料品质和对应采集来源。
- 增加丹药与丹方，覆盖恢复、修炼、战斗准备、突破准备。
- 增加怪物族群、精英、Boss 和对应掉落。
- 补区域掉落、事件奖励和商店 / 储物等必要配置。

验收：

- 新配置都有来源、用途和消耗去向。
- 新丹药不会让旧丹药失效；新怪物不会破坏 P4 难度修正。
- 配置校验和相关回归通过。
- 不新增复杂生产线或新战斗系统。

## 决策记录

- 暂不拆大型项目管理系统；Markdown 看板够用。
- 暂不扩“高阶境界完整内容”；先把练气前期闭环做稳。
- 暂不新增依赖；Godot 场景、GDScript、YAML 和现有测试脚本覆盖当前需要。
