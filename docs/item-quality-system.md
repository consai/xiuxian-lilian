# 道具阶位、品质与颜色

本文档约定道具、法宝、技能展示的两层颜色体系。运行时：

- `res://scripts/enum/enum_item_tier.gd` 负责阶位底盘颜色。
- `res://scripts/enum/enum_quality.gd` 负责品质点、道具名和详情标题颜色。

## 视觉分工

| 维度 | 含义 | 视觉承载 |
| --- | --- | --- |
| 阶位 | 对应境界层级，决定物品基础强度底盘 | 道具格底纹、内描边 |
| 品质 | 同阶内成色，只分下品/中品/上品/极品 | 左上品质点、道具名颜色、详情标题颜色 |

例子：

| 物品 | 展示结果 |
| --- | --- |
| 练气极品 | 一阶底盘色 + 极品红点/红名 |
| 筑基下品 | 二阶底盘色 + 下品褐点/褐名 |

强度排序建议：阶位权重大于品质权重；展示稀有感由品质负责。

```yaml

```

## 配置要求

- 道具配置必须显式提供 `tier` 与 `quality`。
- `tier` 只允许 `1..9`。
- `quality` 只允许 `1..4`，或 UI 侧传入 `下品`、`中品`、`上品`、`极品`。
- 不再兼容旧的 `rarity`、`realm_tier`、`普通/稀有/传说`、`common/rare/legendary`、`品质4` 等写法。

## UI 规则

- 道具格底纹和内描边使用 `EnumItemTier.get_color(tier)`。
- 道具格左上品质点使用 `EnumQuality.get_color(quality)`。
- 道具名称、悬浮提示标题、详情标题使用品质色。
- 背包、奖励结算、GM 面板、物品详情均显示完整档位名，例如 `二阶 · 下品`。
- 当前选中、可点击、不可学习等交互状态不得复用阶位色或品质色，避免混淆。
