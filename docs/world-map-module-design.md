# 世界地图模块设计案

## 1. 模块目标

世界地图用于承载修真世界中的长期移动、城市访问与野外探索入口。

地图节点分为两类：

- **城市类节点**：以单点显示，城市之间通过网状路线连接。
- **野外地图**：以大片区域显示，不绘制固定路线，进入后由独立探索玩法决定移动。
- **野外地点**: 单点表示, 没有路线, 归属于野外地图, 靠近展示(可独立配置范围, 不配置则默认范围),  或者鼠标悬停野外地图1秒后展示, 鼠标移出2秒消失(做缓慢消失和显示)

## 2. 核心体验

玩家打开世界地图后，应快速理解：

1. 当前身处何处。
2. 哪些城市已经发现、可以前往。
3. 城市之间有哪些可用路线。
4. 哪些大片区域属于野外探索范围。
5. 前往目标需要消耗多少时间或资源。

地图保持洞府 UI 的米黄手绘风格。城市路线清晰但不过度抢眼，野外区域以半透明色块和边界表现范围。

## 3. 地图结构

节点状态：
  未发现
  已发现
  已消失

### 3.1 城市类节点

城市类节点只有一个坐标点，组成网状路线。

城市类型：

| 类型 | 用途 | 示例 |
|---|---|---|
| 坊市 | 交易、委托、拍卖、补给 | 青石坊市 |
| 修真城市 | 宗门、居所、传送、综合服务 | 云岚仙城 |

路线状态：

- 未发现路线：不显示。
- 已发现路线：以浅棕色线显示。
- 当前可行路线：以绿色或蓝色高亮。
- 封锁路线：以虚线或红色标记。

城市路线是无向图或有向图边：

```text
云岚仙城 ─ 青石坊市 ─ 落霞镇
    │         ╲          │
  古剑冢 ─── 天河渡口 ──┘
```

### 3.2 野外地图

野外地图以大片区域显示，无固定路线。

野外区域示例：

- 青岚山脉
- 黑水沼泽
- 北境荒原
- 无尽海域

野外区域属性：

- 推荐境界
- 危险等级
- 环境标签
- 主要资源
- 已探索度
- 是否已发现
- 当前世界状态影响

玩家点击野外区域，弹出野外详情弹窗。 区域内部移动、事件和撤退由历练或独立野外模块处理。


### 3.3 野外地点
野外地点:
| 类型 | 用途 | 示例 |
| 特殊事件点 | 剧情、秘境入口、限时事件 | 古剑冢 |

- 推荐境界
- 危险等级
- 环境标签
- 主要资源
- 已探索度
- 是否已发现
- 当前世界状态影响

解锁后 靠近展示




## 4. UI 布局

### 4.1 主地图

- 顶部：标题、当前位置、时间、灵石。
- 左侧：地图图例和筛选按钮。
- 中央：世界地图画布。
  - 城市节点为单点按钮。
  - 城市路线使用 `Line2D`。
  - 野外地图使用大面积 `Polygon2D` 与区域标签。
- 右侧：当前选中目标摘要。
- 底部：地图说明

### 4.2 城市详情弹窗

展示：

- 城市名称与类型。
- 城市描述。
- 城市功能。
- 路程、消耗天数、路线状态。
- “前往城市”按钮。

### 4.3 野外详情弹窗

展示：

- 区域名称、危险度、推荐境界。
- 环境标签与可能收获。
- 探索进度。
- 可选择探索区域: 外围探索, 深入核心, 锁定历练时的难度上下限

### 4.4 野外地点详情弹窗
展示：
- 区域名称、危险度、推荐境界。
- 环境标签与可能收获。
- 探索进度。
- "前往" 按钮  

### 4.5 旅行确认弹窗

展示：

- 起点与终点。
- 路线经过节点。
- 预计消耗天数。
- 可能遭遇提示。
- 确认出发与取消按钮。

## 5. 场景拆分

```text
scenes/map/map.tscn
scenes/map/components/city_map_node.tscn
scenes/map/components/wilderness_region_card.tscn
scenes/map/popups/city_detail_popup.tscn
scenes/map/popups/wilderness_detail_popup.tscn
scenes/map/popups/travel_confirm_popup.tscn
```

场景复用原则：

- 通用物品、技能、背包格等优先复用 `res://scenes/items/`。
- 地图节点与地图弹窗在地图模块内创建可复用子场景。
- 主地图只负责组合，不重复手写相同城市节点和区域卡片结构。

## 6. 管理代码职责

### `world_map_controller.gd`

- 接收地图节点点击。
- 切换城市、野外和旅行确认弹窗。
- 请求世界地图数据。
- 请求旅行和进入野外探索。

### `city_map_node.gd`

- 接收城市节点展示数据。
- 发出城市选择信号。
- 更新节点状态外观。

### `wilderness_region_card.gd`

- 接收野外区域展示数据。
- 发出区域选择信号。
- 更新区域状态外观。

本阶段函数只保留 TODO，不实现完整逻辑。

```json

    "yunlan_city": {
      "name": "云岚仙城",
      "type": "cultivation_city",
      "position": [620, 245],
      "services": ["坊市", "传送阵", "宗门驻地"]
    }
  },
  "routes": [
    {
      "from": "yunlan_city",
      "to": "qingshi_market",
      "days": 2,
      "state": "open"
    }
  ],
  "wilderness_regions": {
    "qinglan_mountain": {
      "name": "青岚山脉",
      "danger": 1,
      "recommended_realm": "炼气一层",
      "exploration": 35,
      "near_city":["yunlan_city"],
      "near_regions":[],
      "sub_location":["mirror_lake"],
    }
  },
  "wilderness_location":{
    "mirror_lake":{
      "name":"镜湖",
```

运行时位置、发现状态和路线状态应经 `DataStore` 管理；静态地图配置建议放入 `res://data/world_map.json`。

## 8. 后续实施顺序

1. 建立 `world_map.json` 与数据校验器。
2. 在 `DataStore` 增加世界地图运行时和存档字段。
3. 实现城市图寻路与旅行消耗。
4. 接入城市功能和野外探索场景。
5. 增加路线事件、封锁和世界状态变化。
6. 增加地图缩放、拖拽和小地图。

## 9. 验收标准

- 城市节点可清晰组成网状路线。
- 野外区域以大片区域表现，且不绘制路线。
- 城市、野外与旅行确认均有独立弹窗。
- 主场景复用地图节点子场景。
- 管理脚本只有 TODO 接口，不包含完整业务逻辑。
- UI 风格与现有洞府、历练界面一致。
