# 修仙数值设计总纲

> 适用范围：角色属性、技能、功法、大道知识、装备法宝、丹药、敌人、历练、突破与经济。
>
> 核心目标：让玩家通过构筑改变解题方式，而不是只比较总战力；让长期成长明显，但不因多套系统互乘而失控。

---

## 1. 数值系统职责

| 系统 | 主要职责 | 允许提供的纵向强度 | 不应承担 |
|---|---|---:|---|
| 境界 | 内容准入、基础盘抬升、新机制解锁 | 高 | 决定一切胜负的硬压制 |
| 根基属性 | 决定角色基础战斗轮廓 | 中 | 提供复杂规则特效 |
| 资质属性 | 决定成长效率、事件路线与长期倾向 | 低 | 直接大量增伤 |
| 大道知识 | 解锁能力、提高专业效率 | 中 | 每级都只给纯数值 |
| 功法 | 定义流派引擎、资源循环与常驻特性 | 中 | 同时成为最佳输出、防御、修炼功法 |
| 技能 | 定义战斗决策与即时行为 | 中 | 永久成长来源 |
| 装备 / 法宝 | 调整构筑、补短板、提供主动能力 | 中 | 无条件取代功法或技能 |
| 丹药 / 消耗品 | 用资源换取短期优势或突破准备 | 低至中 | 无限永久堆属性 |
| 突破品质 | 奖励准备充分与长期经营 | 中 | 形成无法追赶的永久滚雪球 |

### 三类成长

1. **基础盘成长**：境界、根基、装备基础值。负责“变强”。
2. **构筑成长**：功法、技能、法宝、知识组合。负责“变得不同”。
3. **经营成长**：丹药、材料、洞府、关系、突破准备。负责“用积累换机会”。

同一奖励只应主要属于其中一类。一个来源若同时提高攻击、防御、修炼速度、掉落和突破成功率，通常已经越权。

## 2. 统一结算顺序

所有战斗属性统一按以下顺序计算：

```text
基础推导值
+ 境界固定值
+ 装备固定值
+ 功法固定值
+ 其他固定值
= 固定值小计

最终面板 = 固定值小计 × (1 + 同类百分比加值之和)
```

伤害统一按以下顺序计算：

```text
技能原始值 = 对应攻击 × 技能倍率 + 技能固定值
防御减伤率 = 防御 / (防御 + 当前境界防御常数)
基础结算伤害 = 技能原始值 × (1 - 防御减伤率)
最终伤害 = 基础结算伤害 × 暴击 × 易伤 × 最终伤害修正
```

### 乘区限制

只保留四个伤害乘区：

1. 面板攻击；
2. 技能倍率与固定值；
3. 暴击；
4. 最终伤害修正。

知识、功法熟练、灵根、装备词条默认进入同一个“效果加值池”，同类先相加。除标志性终局能力外，不新增独立乘区。

### 百分比护栏

| 数值 | 常规软上限 | 硬上限 |
|---|---:|---:|
| 暴击率 | 40% | 60% |
| 暴击伤害 | 220% | 300% |
| 闪避率 | 35% | 65% |
| 穿透 | 35% | 70% |
| 易伤 | 30% | 60% |
| 冷却缩减 | 25% | 40% |
| 资源消耗降低 | 30% | 50% |
| 吸血 | 15% | 30% |
| 单类减伤 | 50% | 75% |
| 控制成功率 | - | 95% |

## 3. 境界基准与战斗节奏

境界系数用于制作内容预算，不直接作为隐藏伤害倍率。

| 境界 | 内容系数 | 普通战斗目标时长 | 精英战目标时长 | Boss 战目标时长 |
|---|---:|---:|---:|---:|
| 练气 | 1.0 | 8–14 秒 | 20–30 秒 | 45–70 秒 |
| 筑基 | 1.7 | 9–15 秒 | 22–34 秒 | 50–80 秒 |
| 金丹 | 2.8 | 10–16 秒 | 24–38 秒 | 55–90 秒 |
| 元婴 | 4.5 | 10–18 秒 | 26–42 秒 | 60–100 秒 |
| 化神 | 7.0 | 12–20 秒 | 30–48 秒 | 70–110 秒 |
| 炼虚 | 11.0 | 12–22 秒 | 32–52 秒 | 75–120 秒 |
| 合体 | 17.0 | 14–24 秒 | 36–58 秒 | 80–130 秒 |
| 大乘 | 26.0 | 14–26 秒 | 40–65 秒 | 90–145 秒 |
| 渡劫 | 40.0 | 16–28 秒 | 45–72 秒 | 100–160 秒 |

### 同境界强度带

以该境界“标准构筑”为 `100`：

| 对象 | 强度预算 |
|---|---:|
| 弱小敌人 | 55–70 |
| 普通敌人 | 80–95 |
| 强敌 | 105–120 |
| 精英 | 140–170 |
| Boss | 220–300，依靠机制而非纯属性 |
| 刚进入该境界的玩家 | 85–95 |
| 成型专精玩家 | 120–140 |
| 极限构筑 | 不超过 165 |

低一大境界的成型专精玩家可挑战高一境界普通敌人，但不应稳定击败高一境界精英。

## 4. 属性体系

现有四维根基方向合理，建议保持：

| 根基 | 战斗身份 | 非战斗身份 |
|---|---|---|
| 肉身 `body` | 气血、物攻、物防 | 负重、采集、环境承载 |
| 灵力 `spirit` | 法力、法攻、法防 | 阵法驱动、飞行续航 |
| 神识 `sense` | 命中、行动、控制 | 感知、制造精度 |
| 身法 `agility` | 闪避、追逃、位移 | 旅行安全、探索机动 |

资质建议统一为：

| 资质 | 职责 |
|---|---|
| 悟性 `comprehension` | 知识与功法领悟效率 |
| 心性 `will` | 心魔、长期修持、渡劫稳定 |
| 福缘 `fortune` | 稀有事件权重与保底推进 |
| 灵根 `roots` | 元素适配、修炼路线与组合能力 |

### 属性点价值校验

在标准构筑下，投入同等成本后：

- 主属性应带来约 `1.0` 单位价值；
- 合理副属性应带来约 `0.65–0.85`；
- 错配属性仍应带来至少 `0.30`；
- 任一属性若对某流派价值超过其他属性两倍，应拆分其职责或降低系数。

### 速度约束

当前行动间隔公式可继续使用：

```text
行动间隔 = clamp(1.2 × 100 / spd, 0.3, 12.0)
```

但 `spd` 同时提高单位时间内的普攻、技能尝试和资源恢复机会，价值极高。建议：

- 同境界常规 `spd` 差距控制在 ±25%；
- 极限流派最多达到标准角色的 1.6 倍行动频率；
- 控制、爆发和高频流派不可同时拥有顶级速度。

## 5. 技能数值设计

### 技能的六项预算

每个主动技能必须显式分配以下预算：

1. 即时伤害或治疗；
2. 持续效果；
3. 控制；
4. 防护与位移；
5. 资源效率；
6. 施法风险与冷却。

技能不能只按“固定伤害更高”区分。高阶技能至少应改变目标、时机、资源循环或战场规则之一。

### 标准技能模板

以同境界标准角色一次普攻期望伤害为 `1.0A`：

| 技能类型 | 总效果预算 | 冷却 | 资源成本 | 设计用途 |
|---|---:|---:|---:|---|
| 低耗循环技 | 1.3–1.7A | 3–5 秒 | 最大资源 6–10% | 稳定输出 |
| 爆发技 | 2.5–3.5A | 10–16 秒 | 15–25% | 击杀窗口 |
| 终结技 | 4.0–6.0A | 25–45 秒 | 30–50% | 高风险爆发 |
| 轻控制技 | 0.7–1.1A + 控制 | 8–14 秒 | 12–20% | 打断与创造窗口 |
| 强控制技 | 0–0.6A + 强控制 | 16–30 秒 | 20–35% | 改变战局 |
| 防护技 | 吸收 2.0–3.5A | 12–22 秒 | 15–30% | 应对敌方爆发 |
| 位移技 | 0–0.5A + 位移 | 5–10 秒 | 5–12% | 躲避与追击 |

`A` 必须由自动化战斗模拟按当前境界面板计算，不使用手填常量。

### 技能效果换算

用于内部比较，不展示给玩家：

| 效果 | 等效预算 |
|---|---:|
| 1 秒硬控 | 约 0.8A |
| 1 秒沉默 / 缴械 | 约 0.55A |
| 1 秒 30% 减速 | 约 0.20A |
| 自身位移一次 | 约 0.35–0.70A |
| 目标每增加 1 个 | 单目标效果乘 0.65–0.75 |
| 可稳定打断 | 约 0.5A |
| 需要持续施法 | 总预算可提高 15–30% |

### 知识成长

知识等级的主要价值应是解锁与稳定性，纯数值成长建议：

| 知识等级 | 相对刚满足门槛的效果提升 |
|---|---:|
| I | 0% |
| II | 6% |
| III | 14% |
| IV | 24% |
| V | 36% |

首版技能知识成长使用显式 `knowledgeGrowth`，按知识掌握率在 `base` 与
`base + knowledgeGrowth` 之间插值，不再对完整技能效果增加独立乘区。

## 6. 功法数值设计

功法是“构筑引擎”，不是装备栏里的大号属性词条。

### 槽位职责

| 槽位 | 核心职责 | 强度预算 |
|---|---|---:|
| 主功法 | 修炼效率、核心资源循环、流派机制 | 100% |
| 辅助功法 1 | 补强或联动 | 35–45% |
| 辅助功法 2 | 补强或联动 | 35–45% |
| 身法 | 位移、闪避、追逃与先手 | 40–50% |

辅助功法若只生效知识、不生效战斗效果，必须在 UI 中明确；否则玩家会自然认为四个槽位的效果都生效。

### 每部功法的效果结构

每层完整功法建议固定为：

1. **核心效果**：流派主轴，例如雷法暴击后回灵；
2. **保障效果**：解决续航、生存或稳定性问题；
3. **标志效果**：改变规则，例如溢出护盾转化为法力；
4. **代价或约束**：至少一个，例如高耗蓝、怕打断、特定元素限定。

### 功法数值预算

同境界同品质功法，换算后的总收益应接近：

| 效果类型 | 主功法建议预算 |
|---|---:|
| 单一面板属性 | +18–25% |
| 两项相关属性 | 各 +10–15% |
| 资源恢复 | 每 10 秒恢复最大资源 8–14% |
| 修炼效率 | +10–20% |
| 强标志机制 | 占总预算 35–50%，相应减少纯属性 |

功法知识反哺当前最高可再提高 `50%`。建议它只强化标志效果或计入统一效果加值池，不要对全部功法效果再做独立乘法。

## 7. 需要补充的系统

### 7.1 资源体系

除气血、法力外，建议只为明确流派增加专属资源：

| 资源 | 获取 | 消耗 | 目的 |
|---|---|---|---|
| 剑意 | 命中、完美闪避 | 剑技强化 | 鼓励持续进攻 |
| 丹火 | 施法、时间积累 | 炼丹与火系爆发 | 管理节奏 |
| 心境 | 休息、事件、功法 | 抵抗心魔与突破 | 连接战斗外经营 |
| 业力 | 特殊选择累积 | 改变事件、渡劫 | 形成长期代价 |

同一流派最多管理一个战斗专属资源，避免自动战斗规则难以理解。

### 7.2 状态与抗性

状态必须具有统一标签：

```text
硬控：眩晕、冻结、封印
软控：减速、沉默、缴械、减攻
持续伤害：燃烧、中毒、流血
破防：物防降低、法防降低、易伤
特殊：心魔、因果、诅咒
```

同一目标连续受到硬控后获得递减抗性：

```text
第 1 次时长 100%
第 2 次时长 60%
第 3 次时长 30%
随后短暂免疫
```

Boss 不应完全免控；硬控可转换为削韧、打断或短时弱化。

### 7.3 敌人与遭遇

敌人使用“角色模板 + 战斗角色”制作：

| 模板 | 特征 |
|---|---|
| 猛攻 | 高攻击、低防御、明确爆发窗口 |
| 坚守 | 高防御、低速度、可破防 |
| 游斗 | 高速度、低血量、怕必中与范围 |
| 控制 | 高神识、低正面输出、怕突进 |
| 召唤 | 本体较弱、需要处理召唤物 |

每个精英或 Boss 至少有一个清晰弱点。Boss 强度预算中，纯面板最多占 60%，其余来自可学习、可反制的机制。

### 7.4 经济与奖励

所有成长资源必须同时设计来源与消耗口：

| 资源 | 稳定来源 | 波动来源 | 主要消耗 |
|---|---|---|---|
| 灵石 | 常规历练、交易 | 奇遇、稀有掉落 | 装备、修炼、服务 |
| 修为 | 修炼 | 丹药、事件 | 境界推进 |
| 知识经验 | 训练、功法 | 传承、导师 | 知识等级 |
| 材料 | 采集、战斗 | 稀有事件 | 制造、突破 |
| 心境 | 休息、正向事件 | 奇遇 | 突破、抗心魔 |

建议控制：

- 普通历练净收益应为稳定正数；
- 精英历练的期望收益约为普通历练的 `1.5–2.0` 倍；
- 失败损失应让玩家在 `1–3` 次普通历练内恢复；
- 稀有奖励使用保底进度，不鼓励读档刷随机数；
- 永久属性丹药设置境界服用次数、丹毒或边际递减。

### 7.5 突破

突破值七项结构合理，但“准备得越充分，同时品质更高且成功率更高”会形成双重奖励。建议：

- 品质主要决定机制与少量长期成长；
- 成功率主要由准备充分度决定；
- 一品与下品的永久战斗面板差距控制在 `8–12%`；
- 低品质可通过后续洗炼、重铸道基或高成本事件追赶；
- 失败主要消耗时间、材料和状态，不删除知识或核心能力。

## 8. 当前项目数值审计

### 已具备的良好基础

- 四维根基、资质、物法攻防、命中闪避、控制与软减伤已经进入运行时。
- `132` 个大道知识、`91` 层功法、`45` 个能力和 `216` 个效果 ID 已形成完整内容骨架。
- 技能、功法和知识已使用稳定 ID 与统一效果目录，适合继续数据驱动。
- 突破、历练、装备、法宝和消耗品已经有可接入数值总纲的位置。

### 优先处理的问题

1. **技能熟练成长被直接按满值结算**  
   `EffectResolver.resolve_combat_effects()` 当前使用 `(base + masteryGrowth) × knowledge_multiplier`，未读取技能熟练度。`masteryGrowth` 实际成为固定值。

2. **技能额外固定伤害与基础攻击同时生效**  
   所有能力运行时 `power = 1000`，因此伤害为 `对应攻击 + 技能固定值`。技能表应明确记录攻击倍率，避免设计者只看到固定值。

3. **功法槽位说明与运行时不一致**  
   当前 `build_modifiers()` 只结算主功法；辅助功法与身法槽没有参与战斗属性结算。需要明确这是设计选择还是尚未接入。

4. **大量功法百分比效果缺少统一预算**  
   现有功法效果中百分比项远多于固定值项，且知识反哺最多再乘 `1.5`，容易在高境界形成乘法膨胀。

5. **高阶技能主要依靠固定伤害抬升**  
   从练气技能几十点固定伤害到渡劫技能上千点，方向可用，但必须绑定境界标准面板，否则属性成长可能被固定伤害淹没。

6. **旧敌人仍使用 `atk / def` 标尺迁移**  
   迁移兼容适合作为过渡，不适合长期内容制作。新敌人应直接配置物法攻防与角色模板。

## 9. 首版落地顺序

### P0：练气至筑基可测量纵切片

1. 以 `data/balance_v1.json` 定义练气初期、练气成型、筑基初期及敌人基准。
2. 固定种子平衡测试输出胜率、战斗时长与资源余量。
3. 技能配置使用显式 `powerScale` 与 `knowledgeGrowth`；首版不新增技能熟练度存档。
4. 主功法、辅助功法、身法按 `100% / 40% / 40% / 50%` 权重生效。
5. 练气、筑基敌人直接配置物法攻防、命中闪避与控制属性。

### P1：统一预算

1. 将知识、功法熟练、灵根和装备效果并入统一效果加值池。
2. 为技能生成等效预算报告，检查伤害、控制、治疗和位移。
3. 为功法生成总收益报告，检查同境界同品质差异。
4. 新敌人改用物法面板与敌人模板。

### P2：完善经营与长期风险

1. 接入丹毒、心境、业力、保底和资源消耗口。
2. 将突破品质更多转为机制差异，限制永久面板差距。
3. 为生产、探索和治理建立与战斗同等级别的收益基准。

## 10. 平衡验收指标

每次批量改数后至少检查：

| 指标 | 合格范围 |
|---|---|
| 同境界标准构筑对普通敌人胜率 | 70–85% |
| 同境界成型构筑对普通敌人胜率 | 90–98% |
| 同境界标准构筑对精英胜率 | 35–55% |
| 四种主流构筑间最差对局胜率 | 不低于 30% |
| 单一属性极限堆叠相对混合构筑收益 | 不高于 20% |
| 同境界同品质技能预算差 | 不超过 ±15% |
| 同境界同品质功法总收益差 | 不超过 ±12% |
| 一次大境界突破品质造成的永久面板差 | 不超过 12% |
| 战斗结束资源余量 | 常规战保留 25–65% |
| 失败后的恢复周期 | 1–3 次普通历练 |

数值设计不是追求所有选择完全相等，而是让优势有代价、劣势有解法、成长有反馈。
