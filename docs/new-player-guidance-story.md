## 1. 玩家体验目标

玩家是一只刚开始独立修行的小动物修仙者，住在山间小道观。新手引导不再按“一天日课”设计，而是按现有系统的真实节奏走完一次首月闭环：

```text
醒来在清铃观 -> 闭关一个月 -> 服丹炼化 -> 出门巡山
-> 在路线图上选择节点 -> 赢下第一场小战斗 -> 主动返程
-> 回观整理背包 -> 关闭背包 -> 开炉炼丹 -> 进入自由游玩
```

完成后，玩家应自然记住：

- 修炼是“闭关”，最短按月结算，会推动世界时间。
- 丹药能加快修炼，但会带来灵力驳杂，需要巡山战斗压稳。
- 历练已经是路线图玩法：看可前往节点，选择路线，处理节点事件。
- 巡山收获先进入结算和背包，回观后才能稳定使用。
- 背包是独立面板，整理完必须关闭，才能操作丹炉等据点入口。
- 材料能炼丹，丹药又支持下一次闭关。

本阶段不教学大道树、突破、复杂战斗配置、道观建设和长期剧情。

## 2. 设计支柱

| 支柱 | 要求 |
|---|---|
| 可爱但不幼稚 | 主角可以迷糊、认真、尾巴压到书页，但目标清楚 |
| 尊重现有系统 | 不用“一日修炼”“旧历练事件卡”这种过时说法 |
| 少锁入口 | 只引导关键动作，玩家点错时不应卡死 |
| 面板可退出 | 每个跨面板步骤都明确“关闭/返回” |

## 3. 第一章流程：首月闭关

| 步骤 | 剧情节点 | 玩家行为 | 教学目标 | 完成事件 |
|---|---|---|---|---|
| T00 | 风铃醒来 | 阅读开场，看到小道观 | 建立身份和据点 | 进入主界面 |
| T01 | 首月闭关 | 打开修炼界面 | 认识闭关入口 | `tutorial.xiulian_mianban_opened` |
| T02 | 丹药炼化 | 选择丹药炼化并开始闭关 | 理解修炼按月结算 | `tutorial.cultivation_completed` |
| T03 | 出门巡山 | 打开历练入口和地点选择 | 理解巡山目的 | `tutorial.world_map_opened` |
| T04 | 选择旧路 | 选择新手地点并出发 | 认识地点选择 | `tutorial.lilian_started` |
| T05 | 路线图历练 | 在路线图选择节点，直到首战胜利 | 认识路线图和战斗压气 | `tutorial.first_battle_won` |
| T06 | 主动返程 | 点击返程 | 理解保住收获 | `tutorial.lilian_returned` |
| T07 | 回观结算 | 关闭结算回据点 | 理解奖励正式带回 | `tutorial.result_closed` |

| T10 | 自由游玩 | 读结尾 | 明确下一步自选 | 故事完成 |

## 4. 文案基调

开场：

> 山风从清铃观檐角吹过，铜铃叮当响了三声。
> 你从蒲团上坐起来，尾巴还压着半块月课木牌。
> 木牌上写着：先闭关一个月，再下山看看山路。

闭关后：

> 一个月闭关后，聚气丹化开的灵气像热乎乎的云，把经脉塞得鼓鼓的。
> 可云气有些飘，光靠坐着不容易压稳。该去巡山了。

首战后：

> 山路上一场小小争斗后，乱窜的灵气终于安分下来。
> 你拍拍衣摆，把几株灵草收进小背篓。

炼丹前：

> 灵草已经收进背包，先把背包合上。
> 丹炉在观里等着，隔着面板可点不着。

结尾：

> 炉火一跳，第一炉聚气丹滚进瓷盏。
> 小道观里有了药香，也有了下一次闭关的底气。

## 5. 关键机制规格

### 5.1 新局教程状态

**目的**：只让新局进入小兽修首月闭关，旧档不被打断。
**输入**：新游戏、旧存档导入、跳过教程。
**输出**：

- 新局：`tutorial.step = "T00"`，`completed = false`，`skipped = false`。
- 旧档无 `tutorial` 字段：合并为已完成。
- 跳过：`completed = false`，`skipped = true`，入口全部可用。

**失败状态**：旧档进教程，或新局教程不开。
**涉及**：`DataStore`、`GameState.new_game()`、`TutorialService`。

### 5.2 月度闭关

**目的**：让教程与当前修炼系统一致。
**输入**：玩家打开修炼，选择丹药炼化，点击开始闭关。
**输出**：按至少 1 个月结算修为、时间和灵力驳杂。
**失败状态**：文案仍写“1日”“约 200 修为/日”，或引导要求一天修炼。
**涉及**：`cultivation_panel.gd`、`prologue_tutorial.yaml`。

### 5.3 路线图历练

**目的**：适配重做后的历练路线图。
**输入**：玩家进入历练路线图。
**输出**：引导玩家点击可前往节点，遇到事件/战斗正常处理，直到首战胜利。
**失败状态**：只高亮旧 `EventCards`，玩家不知道要点路线图节点。
**涉及**：`expedition_loop.tscn`、`LilianState.settle_pending_battle()`、`prologue_tutorial.yaml`。

### 5.4 背包关闭

**目的**：避免背包面板挡住丹炉导致卡死。
**输入**：玩家打开背包查看首战带回的灵草。
**输出**：下一步先等待 `tutorial.backpack_closed`，再引导 `FurnaceButton`。
**失败状态**：背包未关闭时高亮丹炉。
**涉及**：`beibao_panel.gd`、`TutorialService`、`prologue_tutorial.yaml`。

### 5.5 Esc / 关闭键护栏

**目的**：防止玩家用 Esc 或关闭按钮离开当前教程所需面板后，剧情仍等待面板内事件。
**规则**：

- 等待丹药炼化选择或开始闭关时，修炼面板不可关闭。
- 等待进入新手地点时，世界地图不可返回，野外地点弹窗不可关闭。
- 等待查看首战奖励时，背包不可关闭。
- 等待炼丹步骤完成前，炼丹面板不可关闭。
- 等待 `tutorial.backpack_closed` 时，关闭背包是正确步骤。

**失败状态**：目标节点不在当前界面，教程高亮消失或指向背后按钮。
**涉及**：`TutorialService.is_waiting_for_any()`、各面板关闭处理。

### 5.6 第一炉丹

**目的**：把“材料 -> 丹药 -> 下一次闭关”做成第一个闭环。
**输入**：关闭背包后打开丹炉。
**输出**：默认选中聚气丹方，玩家确认预览后炼制。
**失败状态**：玩家缺材料、缺丹方、找不到按钮。
**涉及**：`AlchemyPanel`、`prologue_tutorial.yaml`。

## 6. 小道观首屏方向

只改气质，不加建设系统。

| 元素 | 表现 |
|---|---|
| 蒲团 | 闭关入口记忆点 |
| 丹炉 | 炼丹入口记忆点 |
| 药架 | 暗示巡山采药 |
| 风铃/晨光 | 清铃观氛围 |
| 山门/石阶 | 巡山入口 |

## 7. 验收

### PM-101

- 新局进入 T00。
- 旧档不会自动进入教程。
- 跳过后修炼、巡山、背包、炼丹入口可用。

### PM-102

- 首战胜利触发 `tutorial.first_battle_won`。
- 引导适配路线图，不要求旧事件卡固定出现。
- 灵草进入历练结算，并在回观后可用于炼丹。
- 查看背包奖励后必须先关闭背包，再引导丹炉。
- Esc / 关闭按钮不会把玩家带到没有当前教程目标的界面。

### PM-103

- 主角默认头像能在角色面板和战斗显示。
- 首战敌人有头像或战斗图。
- 不新增存档字段。

### PM-104

- 主界面从洞府感改成小道观感。
- 文案不再出现“洞府室内”“师父失踪”等旧设定冲突。
- 不改变现有入口逻辑。

## 8. 暂不做

- 道观经营。
- 多动物种族差异。
- 首战分支失败教学。
- 新 UI 框架。
- 为教程单独写一套历练脚本队列。

## 9. 落地顺序

1. 修正教程文本：所有修炼描述改为“闭关一个月”。
2. 历练步骤改为高亮路线图区域，等待首战胜利事件，不绑定旧事件卡。
3. 背包查看首战奖励后新增“关闭背包”步骤。
4. 跑 `run_story_tests.gd` 和场景/历练回归。
5. 再决定是否需要新手专用历练节点；没有试玩证据前先不做。
