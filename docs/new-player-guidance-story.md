# 新手引导与引导剧情设计案

## 1. 设计目标

新手引导的任务不是解释所有系统，而是让玩家在约 12 至 18 分钟内亲手完成一次修仙闭环：

```text
认识洞府 -> 运转功法修炼 -> 查看自身与术法 -> 外出历练
-> 做出第一次抉择 -> 完成首战 -> 主动返程 -> 整理收获 -> 明确长期目标
```

完成引导后，玩家应理解：

- 修炼、历练与突破是长期主循环。
- 气血、法力和时间是需要管理的资源。
- 战斗默认自动进行，但战前配置会影响结果。
- 历练的价值不只是战利品，还会推动事件链和世界状态。
- 青岚山当前存在狼患、古剑冢和魔修活动三条可继续调查的线索。

引导不在首章强制教学炼丹、储物、完整大道树、复杂战斗策略和大境界突破。这些系统应在玩家产生需求时再解锁教学。

---

## 2. 叙事定位

### 2.1 开局钩子

玩家在清风洞府中醒来，师父已经失踪，只留下即将消散的传讯残念与一枚裂开的青玉符。

残念没有能力替玩家完成任何操作，只能在关键时刻给出简短提醒。它既是引导角色，也是贯穿前期的悬念：

> “若你听见此言，我已入青岚山查探异变。莫急着寻我。先运转混元诀，确认自己还能引气入体。”

青玉符会对三类气息产生反应，分别对应项目已有的三条青岚山事件链：

| 玉符反应 | 对应事件链 | 前期悬念 |
|---|---|---|
| 兽血腥气 | `wolf_king` | 狼群为何异常聚集 |
| 残存剑意 | `sword_tomb` | 师父是否进入过古剑冢 |
| 污浊血纹 | `demonic_ritual` | 失踪修士与师父是否被魔修盯上 |

### 2.2 引导角色原则

- 残念只在玩家第一次接触系统或明显停滞时出现。
- 每段文本最多 2 至 3 句，先说处境，再给行动目标。
- 不使用全知口吻。残念可以判断错误，让玩家通过历练发现真相。
- 引导结束后残念消散，青玉符转为长期任务载体，避免永久旁白打扰。

## 3. 第一章：残灯照归途

### 3.1 流程总览

| 步骤 | 剧情节点 | 玩家行为 | 教学目标 | 完成条件 |
|---|---|---|---|---|
| T00 | 洞府惊醒 | 阅读残念，观察界面 | 认识角色状态、日期与当前境界 | 关闭开场对白 |
| T01 | 周天未断 | 点击修炼 | 理解修炼消耗一天并获得修为 | `cultivate` 1 次 |
| T02 | 留下的术法 | 打开人物属性/战斗配置 | 确认混元诀与灵气弹已装备 | 访问配置页并返回 |
| T03 | 山外来信 | 打开世界地图 | 理解洞府和历练入口 | 进入世界地图 |
| T04 | 雾中旧路 | 选择青岚山脉并启程 | 阅读危险度、推荐境界和奖励 | 开始指定引导历练 |
| T05 | 路旁伤者 | 选择是否帮助散修 | 理解抉择、资源消耗与结果 | 完成固定抉择事件 |
| T06 | 青牙狼袭 | 确认迎战并观看首战 | 理解自动战斗、技能与气血法力 | 首战胜利 |
| T07 | 玉符示警 | 选择继续或返程 | 教会风险判断与主动返程 | 点击返程或完成后续保底事件 |
| T08 | 第一份收获 | 查看历练结算并回洞府 | 理解战利品、耗时和世界变化 | 关闭结算页 |
| T09 | 道途初定 | 打开背包，查看所得物品 | 理解物品是成长准备的一部分 | 查看指定战利品 |
| T10 | 残念消散 | 阅读章末对白 | 给出修炼、调查、突破三个长期目标 | 领取章末奖励 |

### 3.2 逐段剧情与交互

#### T00 洞府惊醒

画面从黑场淡入洞府。灯火微弱，打坐蒲团旁悬着一缕残念。

> 师父残念：“清风，洞府禁制尚在，说明追兵没有找到这里。”  
> 师父残念：“先看清自身境界与气血。修行路上，活着比逞强重要。”

交互：

- 高亮顶部境界、气血、法力和日期区域。
- 玩家点击任意空白处后解除遮罩。
- 洞府中仅“修炼”保持高亮，其他功能可点但不强制锁死。

#### T01 周天未断

> 师父残念：“运转一遍混元诀。功法是根，术法只是枝叶。”

玩家点击“修炼”，时间从第 1 日推进到第 2 日，修为增加。

反馈重点：

- 在修为数字旁显示 `+20` 动效。
- 日期变化必须与修为变化同时可见。
- 提示：“修炼会消耗时间；装备的主功法决定修炼效率。”

#### T02 留下的术法

> 师父残念：“我只来得及留下混元诀与一道灵气弹。出门前，确认它们仍在你的行功脉络中。”

玩家打开人物属性或战斗配置页。首次进入时，高亮：

- 主功法槽中的 `method.hunyuan.1`。
- 技能槽中的 `ability.combat.qi_bolt`。
- 自动战斗预设“均衡”。

玩家无需调整配置，查看后即可返回。这样能教会入口，又不会在首轮制造配置负担。

#### T03 山外来信

返回洞府后，青玉符亮起。

> 师父残念：“玉符沾上了青岚山的雾气。若我未归，答案多半还在那里。”  
> 师父残念：“只探外围。气血不足便返程，不要把退路当作耻辱。”

高亮“外出历练”，进入世界地图。

#### T04 雾中旧路

世界地图首次打开时，只对青岚山脉显示任务光标。详情弹窗保留正常信息，并追加：

```text
引导目标：调查青玉符的异动
建议：炼气一层可探索；首次历练会获得基础补给
```

玩家确认启程后进入一局“引导历练”。引导历练仍使用正常历练界面，但前两个遭遇由脚本固定，避免随机事件破坏教学节奏。

#### T05 路旁伤者

固定事件使用现有 `qinglan_injured_cultivator` 的改写版本：

> 一名受伤散修靠在岩边，手中也握着一枚染血青玉碎片。  
> 他认出你的玉符，却在开口前因伤势昏沉下去。

选项：

| 选项 | 即时结果 | 教学作用 |
|---|---|---|
| 渡入法力 | 消耗少量法力；获得一颗回气丹；散修说出“狼群在替人守路” | 教资源交换与善意回报 |
| 先问线索 | 不消耗法力；获得“北面有血色雾气”的情报 | 教选择没有唯一正确答案 |

两条选择都进入首战，不设置错误答案。

#### T06 青牙狼袭

固定触发弱化版青牙狼。战前弹窗中，残念只提示一次：

> “狼在试探你。灵气弹会自动施放，留意气血与法力的变化。”

首战规则：

- 敌人强度保证初始角色在默认配置下稳定获胜。
- 玩家气血低于 40% 时，首次暂停并提示战后应考虑返程或恢复。
- 胜利固定获得灵草与一颗回气丹，确保背包教学有内容。
- 战斗结束突出显示“气血消耗、法力消耗、战利品”三项，不展开复杂战斗日志。

#### T07 玉符示警

首战后青玉符出现三色裂光，远处同时传来狼嚎和金铁轻鸣。

> 师父残念：“这里不只有狼。玉符撑不了多久，你也一样。”  
> 师父残念：“记住今日的分寸。继续，是求机缘；返程，是保住已有收获。”

玩家可以：

- 立即返程：完成教学，保留全部固定奖励。
- 继续前进：下一事件固定为无战斗的“林中残剑”，随后再次提示返程。

这样既教主动返程，也不惩罚想继续探索的玩家。

#### T08 第一份收获

历练结算页首次显示分区提示：

- “耗时”说明历练会推进世界时间。
- “战利品”说明奖励返程后才正式带回。
- “历练纪要”保留刚才的选择和首战记录。
- “世界异动”显示狼患仍在上升，建立长期压力。

#### T09 第一份收获

回洞府后高亮背包。玩家查看灵草或回气丹详情时，提示：

> “资源可以用于恢复、炼丹与突破准备。现在不必记住所有用途，缺少时再来查看。”

不强制装备丹药，不强制打开储物箱。

#### T10 残念消散

> 师父残念：“你已能独自走完一程。接下来，是闭关修炼，还是沿玉符追查，都由你决定。”  
> 师父残念：“若有一日行至炼气九层，别急着筑基。道基高低，会决定你能走多远。”

章末奖励：

- 解锁长期任务“青岚失踪案”。
- 青玉符成为任务物品，不占普通背包容量。
- 可选给予少量灵石或一颗回气丹，不直接赠送高阶战力。

完成后恢复全部正常随机历练规则。

## 4. 引导后的分层教学

后续教学采用“玩家产生需求时触发”，不再使用连续强制步骤。

| 触发条件 | 教学主题 | 引导内容 |
|---|---|---|
| 首次气血低于 50% 并回洞府 | 休息 | 说明休息消耗一天并恢复状态 |
| 首次获得可用丹药 | 道具槽 | 引导将丹药放入战斗道具槽 |
| 首次获得功法或技能典籍 | 学习与配置 | 引导查看前置并替换槽位 |
| 首次历练失败 | 伤势与止损 | 说明失败惩罚、休息和低风险区域 |
| 修为接近当前小层门槛 | 小层提升 | 告知修为达标会自动升层 |
| 炼气九层且达到突破条件 | 大境界突破 | 引导查看七项突破值，不替玩家做选择 |
| 首次打开大道树 | 长期专精 | 只解释训练、前置与专精，不逐节点教学 |

每个分层教学均可从帮助页重看，并允许跳过。

## 5. UI 与交互规则

### 5.1 引导表现

- 使用半透明遮罩、目标描边、短文本卡和方向指示，不移动原有按钮。
- 除必须保证流程的场景跳转外，不硬锁其他功能；玩家点错时给一句上下文提示。
- 同一提示连续失败 2 次后，增加箭头；停滞 20 秒后出现弱提示；不自动替玩家点击。
- 剧情对白可快速点击完成，重开新档时提供“跳过序章”。
- 所有引导奖励在步骤完成时发放，避免玩家因跳过对白损失奖励。

### 5.2 任务追踪

洞府消息区在引导期间改为简短目标：

```text
当前指引：运转混元诀，完成一次修炼
原因：确认经脉与功法仍可正常运转
```

世界地图和历练界面只显示当前一步，不展示完整任务清单，减少信息压力。

## 6. 实现方案

### 6.1 状态模型

在存档增加独立引导状态，不使用 `totals` 推断，以避免旧档、测试档和重复游玩产生误判：

```json
{
  "tutorial": {
    "chapter": "prologue_lost_master",
    "step": "T00",
    "completed": false,
    "skipped": false,
    "flags": {
      "saw_first_combat": false,
      "opened_loadout": false,
      "returned_first_expedition": false
    },
    "seen_context_tips": []
  }
}
```

### 6.2 推荐模块

| 模块 | 职责 |
|---|---|
| `TutorialService` Autoload | 保存步骤、校验完成条件、处理跳过与恢复 |
| `TutorialOverlay` CanvasLayer | 遮罩、高亮、对白、目标提示 |
| `TutorialDirector` | 根据场景和信号推进剧情，不直接持有 UI |
| `data/tutorial_chapters.json` | 文本、步骤、目标节点、完成事件与分支 |

建议通过信号或已有行为结果推进，不在各场景中堆叠大量 `if tutorial_step == ...`：

```text
GameState.cultivate() 完成 -> tutorial_event("cultivated")
SceneManager.go_character_attributes_panel() -> tutorial_event("opened_loadout")
ExpeditionState.start() -> tutorial_event("expedition_started")
ExpeditionState.settle_pending_battle() -> tutorial_event("battle_settled")
GameState.settle_expedition() -> tutorial_event("expedition_returned")
```

### 6.3 引导历练

第一局不要修改正常事件池权重来“碰运气”，应显式支持 scripted queue：

```json
{
  "tutorial_expedition": {
    "location_id": "qinglan_mountain",
    "events": [
      "tutorial_injured_cultivator",
      "tutorial_qinglan_wolf",
      "tutorial_broken_blade"
    ],
    "allow_exit_after": "tutorial_qinglan_wolf"
  }
}
```

队列耗尽、玩家返程或引导被跳过后，立即恢复正常 `ExpeditionDirectorService` 选取逻辑。

### 6.4 兼容与恢复

- 旧存档默认 `tutorial.completed = true`，避免更新后强制进入序章。
- 新档默认从 T00 开始。
- 任意步骤切场景或退出游戏后，可根据 `step` 恢复。
- 若目标节点不存在或事件配置无效，引导应记录错误并自动切到下一安全步骤，不能卡死主流程。
- 跳过引导仍发放首轮最低保障物品，并将青玉符任务正常解锁。

## 7. 数据与内容复用

优先复用现有内容：

- 洞府：`scenes/sim/cave_hub.tscn`
- 人物与战斗配置：现有属性页、功法槽和技能槽
- 世界地图：`scenes/map/map.tscn`
- 历练循环与结算：现有 expedition 场景
- 剧情母题：`qinglan_injured_cultivator`、`qinglan_wolf`、`sword_broken_blade`
- 长期事件链：`wolf_king`、`sword_tomb`、`demonic_ritual`

引导事件使用独立 ID，避免改写正常事件后影响常规平衡和随机池。

## 8. 验收与指标

### 8.1 功能验收

- 新档可从 T00 连续完成至 T10，不依赖随机事件。
- 默认初始配置下首战胜率为 100%，但仍能看见资源消耗。
- 玩家在 T07 返程或继续，都能完成引导。
- 任意步骤退出并重开，能恢复到可继续状态。
- 跳过序章不会漏发关键任务或导致引导事件混入正常事件池。
- 旧档载入后不会自动触发序章。

### 8.2 体验指标

首轮内部测试关注：

| 指标 | 目标 |
|---|---|
| 序章完成时长中位数 | 12 至 18 分钟 |
| T01 前流失 | 低于 5% |
| 能独立说出主循环的玩家比例 | 高于 80% |
| 首次历练知道可以主动返程的玩家比例 | 高于 80% |
| 完成后愿意继续调查任一事件链的玩家比例 | 高于 60% |
| 引导期间打开无关复杂系统的玩家比例 | 用于判断是否需要延迟入口或增加说明 |

### 8.3 测试访谈问题

完成序章后只问四个问题：

1. 你下一步准备做什么，为什么？
2. 修炼和历练分别能给你什么？
3. 什么时候应该返程？
4. 你觉得师父失踪与哪条线索最相关？

若玩家能回答前三题，引导完成了教学任务；第四题用于判断剧情钩子是否有效。

## 9. 制作优先级

| 优先级 | 内容 |
|---|---|
| P0 | 引导状态、洞府高亮、固定首轮历练、首战与返程教学、断点恢复 |
| P1 | 开场/章末对白表现、青玉符长期任务、上下文教学 |
| P2 | 师父残念立绘与音效、分支对白、帮助页回看、埋点统计 |

首个可测试版本应先完成 P0。只要玩家能顺畅走完一次主循环，就可以开始验证；不应等待完整剧情美术后才进行试玩。
