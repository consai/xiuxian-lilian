## realm_balance_service.gd 第 261 行 foundations 校验块已删除
- 原 `collect_config_errors()` 中遍历 `standard_players` 校验 `foundations` 根基层的循环（`var foundations = row.get("foundations", {}) as Dictionary`），因 JSON 把 `foundations` 存成字符串，`as Dictionary` 得 null，`has()` 必崩，校验从未生效。已整体删除该失效块。

## 项目级隐患（待修）：yunxing_params 嵌套对象以 JSON 字符串存储
- `jingjie_balance_standard_play.json` 的 `foundations`、`jingjie_balance_benchmark_ene.json` 的 `attrs`、`jingjie_balance_combat_attrib.json` 的 `scale` 等字段，导出时是被包成字符串（如 `"{\"body\": 10}"`）而非原生 JSON 对象。
- `JsonLoader._export_keyed_rows()` 只剔除 null，不解析嵌套 JSON 字符串；`_coerce_export_scalar()` 也只处理数字/布尔。
- 影响：`build_base_combat_attrs` 中 `scale` 的 `is Dictionary` 判断失败 → 所有 scale 系数被静默忽略，战力公式退化为基础值；`build_standard_player_attrs` 同样丢失 scale。
- 修复方向：在导出脚本中输出原生对象，或在 JsonLoader 加载时解析 JSON 字符串。
